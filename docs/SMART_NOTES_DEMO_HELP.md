# Smart Notes Demo Help

这份帮助只服务 Smart Notes Demo。第一次打开时，不需要先学完整套方法论。照着导览走一遍，你会看到研思录怎样把一条完整路径连起来：

记录 -> 转述 -> 永久笔记 -> 建立关联 -> 主题索引 -> 写作中心

这套 Demo 想表达的核心不是“记得更多”，而是：

**让笔记长出思想。**

更具体地说，就是：

- 认真记录，先抓住值得处理的材料
- 用自己的语言重述，确认自己真的理解
- 沉淀为永久笔记，留下愿意承担的判断
- 通过关联形成知识网络
- 让知识网络持续复利，为写作与创新提供基础

## 第一次打开先走六步

1. 打开“手机上先记一句：我总是收藏很多但不会用”。
   它是一条还没处理完的随笔，展示“记录只是入口”。
2. 打开“阅读一开始就要面向未来写作”。
   它展示材料怎样先被转述，而不是直接复制。
3. 打开“写作不是最后一步，而是整理笔记的方向”。
   它是一条已经确认的永久笔记，展示什么叫“自己的判断”。
4. 打开“待关联练习：保存关系前先写清楚为什么”。
   去建联面板里为它找相邻笔记，并补一句为什么相关。
5. 打开“为什么要关联笔记？”或“永久笔记是什么？”。
   看主题索引怎样围绕问题组织关键笔记。
6. 打开写作项目《为什么研思录要把卡片笔记写作法做进产品里》。
   看写作中心怎样从主题和关键笔记生成提纲。

## 产品核心理念

研思录不是把“记录、整理、关联、写作”做成几个孤立功能，而是把它们连成一条会不断产出新理解的路径。

它想帮助用户做到的是：

- 不是只保存资料，而是留下自己的判断
- 不是只画很多连线，而是说明为什么这些笔记应该放在一起思考
- 不是等最后才写作，而是让前面的记录、转述和关联都为写作做准备
- 不是追求收藏数量，而是让旧笔记在新问题中继续产生价值

## 最佳实践

可以把这套方法先记成一句话：

**记录，重述，关联，让笔记长出思想。**

如果拆成日常动作，就是：

1. 记下一条值得处理的材料或闪念
2. 用自己的语言重述它，不直接拿摘录当结果
3. 把真正成熟的判断沉淀为永久笔记
4. 至少为新笔记建立一条有理由的关联
5. 当 3-7 条笔记围绕同一个问题时，建立主题索引
6. 从主题索引进入写作中心，把已有判断组织成提纲

## 什么是好的永久笔记

好的永久笔记不是资料存放格，而是一条以后还愿意继续使用、修改或反驳的判断。

它至少应该回答：

- 我在说什么判断？
- 为什么我这样判断？
- 这条判断的边界是什么？
- 它和哪些旧笔记有关？

## 什么是好的关联

好的关联要回答：

**这两条笔记放在一起，会让哪个判断更清楚？**

常见关系包括：

- 支持
- 补充
- 限定
- 反驳
- 例子
- 前提
- 桥接

保存前先写一句“为什么要这样连”。没有理由的连线只会让图谱变热闹，不会让思考更清楚。

## 什么是主题索引

当 3 到 7 条永久笔记围绕同一个问题、并且已经有关系理由、边界或反例时，就可以建立主题索引。

主题索引不是文件夹，也不是文章草稿。它更像：

- 下次继续思考这个问题时的入口
- 一组关键笔记的阅读顺序
- 进入写作中心前的整理台

## AI 在这里做什么

AI 可以帮助你发现候选关系、可写主题和提纲入口，但最终判断是否保存，必须由你确认。

Demo 始终保持这个原则：

**AI 给建议，你做判断。**
