# Smart Notes Demo Help

这份帮助只服务 Smart Notes Demo。它解释新用户第一次进入 Demo 时应该先做什么，以及如何理解关联笔记、写作时机和索引笔记。

## 三步走完整流程

1. 先看一条材料如何变成永久笔记：从 `SRC-SMART-NOTES` 到 `FN-SN-001`、`LIT-SN-001`，最后到 `PN-SN-001`。
2. 再处理一条未关联笔记：为一条永久笔记选择相关旧笔记，选择关系类型，写一句理由并保存。
3. 最后进入写作中心：从 `WP-SN-PM-001` 看相关永久笔记如何形成提纲、草稿骨架和文章。

## 什么是关联笔记

关联笔记不是把两个标题放在一起，也不是因为标签相同就连线。它要回答：

- 这条笔记支持了哪条判断？
- 它反驳或限制了哪条判断？
- 它是否补上了两条笔记之间的中间台阶？
- 它能不能帮助以后写作时组织一个段落？

一条关系至少要有一句理由。没有理由的线只会让图谱变热闹，不会让思考更清楚。

## 什么时候从笔记进入写作

不要等资料全部读完才开始写。更好的时机是：

- 已经有 3 到 7 条永久笔记围绕同一个问题；
- 至少有一条支持关系；
- 至少有一条边界、反例或张力；
- 还有一个值得继续追问的问题。

达到这个状态，就可以先生成提纲。提纲不是最终文章，而是用来检查：哪些判断已经站住，哪些地方还缺证据。

## 如何建立索引笔记

索引笔记不是文件夹。它是一张进入主题的入口卡。

建立索引笔记时，只做四件事：

1. 写出一个中心问题。
2. 选出 5 到 9 条最关键的永久笔记。
3. 按阅读或论证顺序排列这些笔记。
4. 给每条笔记写一句“为什么放在这里”。

在 Demo 里，可以先看：

- `IC-SN-001`
- `IC-SN-005`
- `IC-SN-010`

它们展示了索引卡如何把一组永久笔记组织成可重新进入的主题路径。

## 后续如何使用索引笔记

以后遇到一个写作问题，不要从全部笔记里乱翻。先打开相关索引笔记：

- 顺着中心问题读关键笔记；
- 看哪些关系已经清楚；
- 找出缺少证据、反例或桥接的地方；
- 把索引笔记当作文章提纲的种子。

索引笔记的价值不在于把笔记分类，而在于让你下次能快速回到一个问题现场。

## AI 在这里做什么

AI 可以帮你发现可能相关的旧笔记，也可以提示关系理由的草稿。但最终是否保存关系，必须由用户确认。

Demo 中应始终保持这个原则：AI 给建议，用户做判断。
