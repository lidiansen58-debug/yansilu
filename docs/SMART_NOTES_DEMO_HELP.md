# Smart Notes Demo Help

这份帮助只服务 Smart Notes Demo。第一次打开时，不需要先学完卢曼方法，也不需要理解所有字段。照着下面这条路走一遍，就能看到研思录怎样把材料变成判断、关系、主题索引笔记和文章提纲。

## 第一次打开先走五步

1. 看一条材料如何变成永久笔记：打开 `SRC-SMART-NOTES`，再看 `FN-SN-001`、`LN-SN-001` 和 `PN-SN-001`。你要看的不是格式，而是一本书里的想法怎样先被转述，再变成“我愿意承担的一句话判断”。
2. 让一条未关联笔记进入关系网：打开 `PN-SN-101`。找一条真正相关的旧笔记，选择它们的关系，写一句“为什么要这样连”，再保存。保存以后，它就不只是单独放着的一张卡片，而是能和旧判断互相解释。
3. 看多条永久笔记如何形成主题索引笔记：打开 `IC-SN-001`、`IC-SN-005` 或 `IC-SN-010`。先读中心问题，再按顺序读里面列出的永久笔记。主题索引笔记不是文章，它只是一个入口，帮你下次重新进入同一个问题。
4. 从主题索引进入写作中心生成提纲：打开 `WP-SN-PM-001`，再看 `DS-SN-PM-001`。重点看主题索引笔记和永久笔记怎样被放进写作项目，形成提纲和草稿骨架。
5. 做一次定期回顾：检查有没有孤立笔记、过宽标签、没有理由的关系，以及已经可以写成文章的主题。回顾的目的不是大整理，而是决定下一步最该补哪一个小动作。

## 卢曼方法在这里怎么落地

卢曼式卡片盒的核心，不是把笔记写得漂亮，也不是把分类做得很细。它要求每条新判断都尽快进入已有网络，并且让旧笔记在新问题里重新变得有用。

在研思录里，这件事被拆成几个能照做的动作：

- 来源材料只负责说明边界：这条材料从哪里来，准备用来回答什么问题。
- 文献笔记负责用自己的话转述：先确认你真的理解了材料。
- 永久笔记负责沉淀判断：写成以后离开原文也能被重新使用。
- 关系负责说明连接理由：两条笔记为什么应该放在一起看。
- 主题索引笔记负责提供入口：围绕一个中心问题，把几条永久笔记排成可重新进入的路径。
- 写作中心负责把已有判断组织成提纲：不是从空白开始求灵感，而是从已经确认的笔记开始写。

## 什么是好关系

关联笔记不是把两个标题放在一起，也不是因为标签相同就连线。它要回答一个很朴素的问题：这两条笔记放在一起，能让哪个判断更清楚？

保存关系前，先写清其中一种动作：

- A 支持 B；
- A 反驳或限制 B；
- A 给 B 补了一个例子、边界或中间台阶；
- A 和 B 可以一起帮助以后写一个段落。

没有理由的线只会让图谱变热闹，不会让思考更清楚。

## 什么时候建立主题索引笔记

不要等资料全部读完才建立主题索引。更好的时机是：

- 已经有 3 到 7 条永久笔记围绕同一个问题；
- 至少有一条支持关系；
- 至少有一条边界、反例或张力；
- 还有一个值得继续追问的问题。

达到这个状态，就可以先建一张主题索引笔记。它不是文章，也不是最终结论，而是一个入口：下次你想继续这个问题时，可以从这里重新进入。

## 如何定期回顾

每周或每隔几天，用 10 分钟看四类东西就够了：

1. 孤立笔记：有没有永久笔记还没有进入任何关系？先给它找一条最重要的旧笔记。
2. 宽标签：有没有标签大到什么都能放进去？把它改成更具体的问题，或改用主题索引笔记承接。
3. 缺理由关系：有没有关系只说明“相关”，却没说明为什么？补一句理由，或删掉这条关系。
4. 可写主题：有没有一个主题索引笔记已经有中心问题、关键笔记、边界和反例？把它送进写作中心生成提纲。

回顾不是为了把系统收拾整齐，而是为了让下一步动作变得清楚。

## AI 在这里做什么

AI 可以帮你发现可能相关的旧笔记，也可以给出关系理由草稿。但最终是否保存关系，必须由你确认。

Demo 中始终保持这个原则：AI 给建议，你做判断。
