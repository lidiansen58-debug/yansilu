# Smart Notes Demo Help

这份帮助只服务 Smart Notes Demo。第一次打开时，不需要先学概念，按下面 6 步走一遍就够了。

## 第一次按 6 步走

1. 写一条永久笔记：打开 `SRC-SMART-NOTES`，再看 `FN-SN-001`、`LN-SN-001` 和 `PN-SN-001`。重点不是格式，而是材料怎样先被转述，再变成你愿意承担的一句话判断。

2. 立即找一条旧笔记建立关系：写完新永久笔记后，不要先去看大图。先找一条最相关的旧笔记，可以用搜索，也可以让 AI 帮你列出可比较的旧笔记。

3. 给关系写理由：选择支持、限定、反驳、补充或桥接等关系类型，再写一句“为什么这两条要放在一起”。没有理由的连线只是热闹，有理由的连线才会让笔记进入关系网。

4. 创建主题索引笔记：当 3 到 7 条永久笔记围绕同一个问题时，创建一张主题索引笔记。它不是文章，也不是文件夹，而是下次重新进入这个问题的入口。

5. 从主题索引进入写作中心：打开 `WP-SN-PM-001`。看主题索引和相关永久笔记怎样变成文章提纲、草稿骨架和最终文章。

6. 每周回顾一次：检查孤立笔记、太宽的标签、没有说明理由的关系，以及已经可以进入写作的主题。回顾不是大扫除，而是把还没接上的判断继续接回网络。

## 一条材料如何变成永久笔记

材料先不用急着写成文章。先做三件事：

1. 保留来源边界：这条材料从哪里来，哪部分是原文，哪部分是你的理解。
2. 用自己的话转述：如果只能复制原句，说明还没真正吸收。
3. 写成一句判断：这句话离开原材料后也能独立回答一个问题。

在 Demo 里，`SRC-SMART-NOTES` 是来源，`LN-SN-001` 是转述，`PN-SN-001` 是永久笔记。顺着这三张看，就能看到“材料 -> 转述 -> 判断”的落地路径。

## 一条笔记如何进入关系网

写完永久笔记后，马上问：它最应该和哪一条旧笔记放在一起看？

一条值得保存的关系，至少要说清楚一种动作：

- A 支持 B；
- A 反驳或限制 B；
- A 给 B 补了一个例子、边界或中间台阶；
- A 和 B 可以一起帮助以后写一个段落。

如果你只能说“它们都和同一个主题有关”，先不要急着强连。先写下还需要比较的问题，等理由变清楚后再保存关系。

## 什么是主题索引笔记

主题索引笔记不是文章。它还不需要完整开头、结尾和段落。

它也不是普通分类。分类只告诉你“这些东西放在哪里”，主题索引要回答“下次我从哪个问题重新进入”。

创建主题索引笔记时，只做四件事：

1. 写出一个中心问题。
2. 选出 3 到 7 条最关键的永久笔记。
3. 按阅读或论证顺序排列这些笔记。
4. 给每条笔记写一句“为什么放在这里”。

在 Demo 里，可以先看 `IC-SN-001`、`IC-SN-005`、`IC-SN-010`。它们展示了索引笔记如何把一组永久笔记组织成可重新进入的主题路径。

## 什么时候进入写作中心

不要等资料全部读完才开始写。更好的时机是：

- 已经有 3 到 7 条永久笔记围绕同一个问题；
- 至少有一条支持关系；
- 至少有一条边界、反例或张力；
- 每条关键关系都写了理由。

达到这个状态，就可以从主题索引进入写作中心先生成提纲。提纲不是最终文章，而是用来检查：哪些判断已经站住，哪些地方还缺证据。

## 每周回顾看什么

每周只看四类东西：

- 孤立笔记：有没有一条新永久笔记还没有接入关系网？
- 宽标签：有没有一个标签太大，里面其实混着多个问题？
- 缺理由关系：有没有连线只说“相关”，却没说清支持、限定、反驳或桥接？
- 可写主题：有没有一张主题索引已经足够进入写作中心？

这就是卢曼方法在研思录里的落地方式：每次只做一个小动作，但每个动作都会让下一次阅读、连接和写作更容易。

## AI 在这里做什么

AI 可以帮你发现可能相关的旧笔记，也可以给出关系理由草稿。但最终是否保存关系，必须由你确认。

Demo 中始终保持这个原则：AI 给建议，你做判断。
