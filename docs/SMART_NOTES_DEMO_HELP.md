# Smart Notes Demo Help

这份帮助只服务 Smart Notes Demo。第一次打开时，不需要先学完卡片笔记法。照着导览走一遍，就能看到研思录怎样把“记录 -> 转述 -> 永久笔记 -> 建立关系 -> 主题索引 -> 写作中心”连成一条路。

## 第一次打开先走六步

1. 打开“手机上先记一句：我总是收藏很多但不会用”。它是一条还没处理完的随笔。
2. 打开“阅读一开始就要面向未来写作”。它展示文献笔记怎样用自己的话转述材料。
3. 打开“写作不是最后一步，而是整理笔记的方向”。它是一条已经确认的永久笔记。
4. 打开“待关联练习：保存关系前先写清楚为什么”。为它找一条相邻笔记，选择关系类型，并写一句理由。
5. 打开“为什么要关联笔记？”或“永久笔记是什么？”。主题索引会告诉你按什么顺序读关键笔记。
6. 打开写作项目“为什么研思录要把卡片笔记写作法做进产品里”。看写作中心怎样从主题索引和永久笔记生成提纲。

## 卡片笔记法在这里怎样落地

- 随笔只负责抓住还不稳定的想法。
- 文献笔记负责用自己的话转述材料。
- 永久笔记负责沉淀一个可承担的判断。
- 关系负责说明两条笔记为什么要放在一起看。
- 主题索引负责围绕一个问题组织关键笔记。
- 写作中心负责把已确认判断整理成提纲。

研思录的重点不是在外部文档里讲方法论，而是把这些动作做进产品路径里。

## 什么是好关系

好关系要回答：这两条笔记放在一起，会让哪个判断更清楚？

常见关系包括：支撑、补充、限定、反驳、例子、前提、桥接。保存前先写一句“为什么要这样连”。没有理由的线只会让图谱变热闹，不会让思考更清楚。

## 什么时候建立主题索引

当 3 到 7 条永久笔记围绕同一个问题，并且已经有关系理由、边界或反例时，就可以建立主题索引。

主题索引不是文章，也不是文件夹。它是下次继续思考时的入口。

## AI 在这里做什么

AI 可以帮你发现候选关系、缺口和提纲。但最终是否保存，必须由你确认。

Demo 始终保持这个原则：AI 给建议，你做判断。
