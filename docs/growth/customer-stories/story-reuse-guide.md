# Story Reuse Guide

这份指南帮助团队把 `customer-stories` 里的长故事改写成不同形式的对外内容。原则是：先保留“人物和转变”，再根据渠道缩短长度。

## 1. 长故事

适用场景：

- 官网案例页
- 深度公众号文章
- B 站 / YouTube demo 脚本
- 内测招募文章
- 销售材料中的客户场景页

推荐结构：

1. 用一个具体时间或场景开头。
2. 展示用户原来的混乱状态。
3. 点出旧工具为什么没有真正解决问题。
4. 让言思录以一个小动作介入：转述、一句话论点、三句话压缩、主题索引、写作脚手架。
5. 展示用户获得的不是“更多笔记”，而是“更清楚的判断和结构”。
6. 用一句可传播的总结收尾。

## 2. 官网短案例

适用场景：

- landing page 的 use case 模块
- 产品页中的场景卡片
- waitlist 页面

压缩模板：

```text
{角色} 常常遇到的问题不是 {表层问题}，而是 {深层问题}。

言思录帮助他们把 {原始材料} 转化为 {观点单元}，再组织成 {可输出结构}。

结果是：{用户结果}。
```

示例：

```text
内容创作者常常遇到的问题不是没有素材，而是素材没有变成自己的判断。

言思录帮助他们把摘录和灵感转化为一句话论点、三句话压缩和主题索引，再组织成文章脚手架。

结果是：写作不再从空白页开始。
```

## 3. Demo 开场

适用场景：

- 3 分钟产品演示
- 销售演示
- 内测 walkthrough

压缩模板：

```text
今天我们不展示所有功能，只展示一个关键转变：

一个 {角色}，原来有很多 {材料}，但始终卡在 {痛点}。

我们会看到言思录如何把一段材料变成转述，把转述变成一句话论点，再把几条论点组织成可以写作或决策的结构。
```

示例：

```text
今天我们不展示所有功能，只展示一个关键转变：

一个研究生，原来有很多论文、高亮和摘要，但始终卡在文献综述的结构上。

我们会看到言思录如何把一段文献变成转述，把转述变成一句话判断，再把多条判断组织成一个综述主题。
```

## 4. 小红书 / X / LinkedIn 短帖

适用场景：

- 痛点触发
- 场景种草
- 创始人叙事
- 产品教育

短帖结构：

1. 先说一句反常识判断。
2. 给一个具体场景。
3. 点出用户误以为的问题。
4. 给出真正问题。
5. 引出言思录的转变。

示例：

```text
记了很多笔记却写不出来，不一定是你不够自律。

很多内容创作者的问题不是没有素材，而是素材一直停留在“我看过”，没有进入“我判断”。

摘录不能直接变成文章。它中间至少要经过转述、一句话论点、三句话压缩和主题组织。

言思录想补的就是这段路：从资料到观点，再从观点到写作结构。
```

## 5. 销售话术

适用场景：

- 用户访谈
- B2B 销售
- 内测沟通
- 合作介绍

一句话版本：

```text
言思录不是帮你存更多内容，而是帮你把已有内容加工成可追溯的判断和可输出的结构。
```

按角色改写：

- 对创作者：帮你把收藏和摘录变成观点库，写作不再从空白页开始。
- 对研究者：帮你把文献笔记变成问题结构和综述脚手架。
- 对产品经理：帮你把用户反馈变成可追溯的产品判断。
- 对咨询顾问：帮你把访谈、数据和竞品材料变成报告主张。
- 对创业者：帮你把客户访谈和市场观察沉淀成战略判断。

## 6. 统一表达边界

对外讲故事时，建议坚持这些边界：

- 不说言思录“一键写完文章”。
- 不把 AI 描述成替代用户判断的角色。
- 不把产品讲成普通资料仓库。
- 不只讲双链、图谱、Markdown 等工具细节。
- 要反复强调从资料到观点、从观点到结构、从结构到输出。

## 7. 最适合优先传播的三条故事线

第一优先：

> 内容创作者：记了很多，却写不出来。

理由：痛点直观，容易传播，转化路径短。

第二优先：

> 研究者：文献很多，但综述不是文献排列。

理由：付费意愿和使用频率较高，产品差异化清晰。

第三优先：

> PKM 重度用户：链接很多，但观点没有成型。

理由：容易进入现有知识管理社区，也能清楚区分言思录和传统笔记工具。
