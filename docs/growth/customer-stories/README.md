# Customer Stories for Yansilu

这个目录用来沉淀言思录的客户故事。这里的故事不是功能清单，也不是传统敏捷开发里的短句式 user story，而是面向客户理解、官网表达、demo 讲述、销售沟通和内容传播的“场景化故事库”。

这些故事默认是 fictional composite stories：它们不是某个真实用户的逐字案例，而是基于目标客户画像、产品定位和典型痛点写成的复合型故事。对外使用时，可以改写成更短的官网模块、短视频脚本、销售话术或 demo 旁白。

## 核心叙事

言思录最值得讲的故事，不是“我有一个笔记工具”，而是：

> 我明明读了很多、摘了很多、收藏了很多，却仍然在真正写作、研究和判断时从零开始。言思录帮助我把资料加工成自己的观点，再把观点组织成可以继续生长的写作结构。

## 每个故事的结构

每个故事尽量包含以下元素：

1. 人物是谁：让客户看到“这就是我”。
2. 某个具体时刻：让痛点落在一个真实场景里。
3. 旧工作流如何失败：让问题变得可感知。
4. 言思录如何介入：展示产品动作，但不写成说明书。
5. 转变发生在哪里：从资料、笔记、观点、结构到输出。
6. 一句可传播的结论：方便拿去做官网标题、视频标题或销售开场。

## 可复用短句

- 不是帮你记更多，而是帮你想得更清楚。
- 把资料变成观点，把观点变成文章。
- 从收藏夹到观点库，从空白页到写作脚手架。
- 你的笔记不该只是仓库，它应该成为思考的工作台。
- AI 辅助思考，但不替你思考。
- 写作不从空白页开始，而从已经被提炼过的理解开始。
