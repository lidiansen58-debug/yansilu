# 研思录 Product Spec v1.3

## 1. 文档目的

本版本用于统一研思录 v1 的核心方向、术语、信息架构与交互边界，确保研发、设计、测试和 AI 编排在同一约束下推进。

核心目标不是“记录更多”，而是让用户更快完成：
`资料输入 -> 书摘笔记 -> 原创笔记 -> 索引与连接 -> 图谱思考 -> 写作篮 -> 提示词与提纲（MVP） -> 原创成文`

---

## 2. 术语与命名（统一）

1. Vault（技术名）= 目录工作区（UI显示为“目录”）
2. 闪念笔记（旧称）= 随笔记录（UI显示）
3. 文献笔记（旧称）= 书摘笔记（UI显示）
4. 永久笔记（旧称）= 原创笔记（UI显示）

说明：
- 领域模型与 API 可继续使用 `fleeting/literature/permanent` 作为内部类型标识。
- 前端展示和产品文案统一使用“随笔记录/书摘笔记/原创笔记”。
- 面向普通用户的文件组织概念统一叫“目录”，不再在界面主流程中强调“卡片盒”。内部仍可使用 `CardBox/Directory` 表示带元数据的本地目录。

---

## 3. 目录模型（信息架构）

## 3.1 根目录与默认目录

1. 系统必须支持用户设置工作区根目录。
2. 系统必须提供默认目录树：
- 随笔目录（存放随笔记录）
- 书摘目录（存放书摘笔记）
- 原创目录（存放原创笔记）
3. 三类笔记分别存在于不同目录树下，创建笔记时由当前入口与当前目录决定类型。

## 3.2 子目录规则

1. 可在默认目录下新建子目录，用于细分不同主题/领域。
2. 子目录支持删除、隐藏。
3. 新建目录可配置最大笔记数量，默认值 `500`。
4. 每个原创目录有独立索引区。
5. 新建目录时必须支持通过系统目录选择器浏览并选择本机保存位置。

## 3.3 写作入口约束

1. 只有“原创目录”中的原创笔记可进入写作篮与脚手架流程。
2. 书摘笔记和随笔记录不得直接绕过原创笔记层进入成稿流程。

---

## 4. v1 核心模块（P0）

1. 目录中心（替代难理解的 App Shell 表达）
2. 随笔记录
3. 书摘笔记工坊
4. 原创笔记编辑器
5. 索引中心
6. 图谱工作台（本地/主题/写作）
7. 写作中心（写作项目、写作篮、脚手架）
8. 设置与导入导出

---

## 5. 模块约束

## 5.1 目录中心

职责：
- 路由管理与全局导航
- 全局搜索入口
- 命令面板
- 最近项目/最近笔记/最近索引入口
- 三类默认目录切换与隐藏目录管理

## 5.2 随笔记录

职责：
- 快速记录文字与语音
- 标记待处理内容
- 一键转换为原创笔记或书摘笔记

行为规则：
1. 支持文本与语音输入。
2. 超过 7 天且未处理的随笔默认隐藏（可通过筛选显示）。
3. 新建且未处理条目显示“新建”标记。
4. 随笔可转换为原创笔记，也可转换为书摘笔记。
5. 转换完成后，原随笔自动归档，并保留转换链路。

## 5.3 书摘笔记工坊

职责：
- 录入来源并形成书摘结构
- 管理摘录与转述
- 从书摘直接发起原创笔记

书摘笔记字段（P0 最小集）：
1. 作者姓名
2. 出版年份
3. 书名（主/副标题）
4. 出版社
5. 页码
6. 版本信息（可选）
7. 译者/编者信息（适用时）
8. 摘录文字
9. 转述（paraphrase）

关键约束：
- 没有页码定位或没有转述时，不可完成保存。
- 可在书摘笔记中直接“新增原创笔记”，并在原创笔记内管理该书摘引用。
- 书摘笔记可以长期单独存在；从书摘新增原创笔记后，书摘保持原状态，不自动归档或改为已转换。
- 书摘笔记可以链接到一个或多个原创笔记，作为引用来源长期复用。

## 5.4 原创笔记编辑器

职责：
- 编辑核心观点、理由、边界/反例
- 管理来源引用、索引归属、笔记关联
- 接收原创性守卫反馈

关键约束：
1. 不提供“AI 一键生成原创笔记正文”。
2. 粘贴大段原文需提示风险。
3. 保存时触发 originality guard。
4. 与来源内容相似度 `>= 80%` 时禁止保存文件。
5. 标题永远取 Markdown 正文第一行。
6. 关联笔记正文显示为 `[[标题]]`，内部通过 SQLite 绑定 `note_id`。
7. 标签使用正文中的 `#标签`，解析后的标签关系写入 SQLite。

## 5.4.1 快捷键（Windows/Linux）

说明：
- Mac 默认映射：`Ctrl -> Cmd`，`Alt -> Option`。
- 允许用户在设置中重绑快捷键。

写作编辑场景：
1. 加粗：`Ctrl+B`
2. 斜体：`Ctrl+I`
3. 高亮：`Ctrl+Shift+H`（默认可配）
4. 行内代码：`Ctrl+\``
5. 数学公式：`Ctrl+/`
6. 编辑/阅读模式切换：`Ctrl+E`
7. 源码模式切换：`Ctrl+R`

结构化写作场景：
1. 选中多行转无序列表：`Ctrl+L`
2. 一级到六级标题：`Ctrl+1 ... Ctrl+6`
3. 引用块：`Alt+Q`（默认可配）
4. 分割线：`---` + 回车（文本规则）

检索与导航场景：
1. 当前笔记搜索：`Ctrl+F`
2. 全局搜索：`Ctrl+Shift+F`
3. 查找下一个：`F3`（避免与图谱冲突）
4. 查找上一个：`Shift+F3`
5. 反向链接面板：`Ctrl+Shift+O`
6. 关系图谱：`Ctrl+Shift+G`

## 5.4.2 编辑器工具栏与菜单（按钮化）

原创笔记编辑器必须提供可点击按钮，不要求用户只靠 Markdown 输入。

一级菜单：
1. 文件：保存、另存、导出 Markdown
2. 编辑：撤销、重做、查找、替换
3. 插入：标题、列表、任务、引用、代码、公式、分割线
4. 关联：关联笔记、反向链接、关系类型
5. 标签：新增标签、管理标签、按标签筛选
6. 视图：编辑/阅读/源码模式，侧栏开关

高频工具栏按钮：
1. `B`（加粗）
2. `I`（斜体）
3. `H`（标题）
4. `"`（引用块）
5. 列表图标（列表）
6. 链接图标（插入 `[[标题]]`）
7. 标签图标（插入或选择 `#标签`）
8. 反向链接/关联图标
9. 原创性检测图标
10. 保存图标

视觉约束：
- 工具栏应采用轻量图标按钮，不使用大号表单按钮。
- 默认无边框或弱边框，hover 时才显示浅色背景。
- 工具组之间用细竖线分隔。

## 5.4.3 关联笔记与标签行为（强约束）

1. 点击“关联笔记”按钮，等效于在光标处输入 `[[` 并打开候选面板。
2. 候选检索默认范围为“当前原创目录 + 原创笔记类型”。
3. 点击“标签”按钮，等效于插入 `#` 并弹出标签候选/新建面板。
4. 点击任意标签链接后，展示“当前目录内、拥有该标签的全部原创笔记”。
5. 标签结果页必须支持：
- 按更新时间排序
- 按子目录二次筛选
- 一键回跳到原笔记

## 5.5 原创笔记关联交互（参考 Obsidian）

1. 输入 `[[` 触发候选联想（标题/别名/标签）。
2. 支持就地新建目标笔记并插入链接。
3. 悬停可预览，侧边可打开目标笔记。
4. 保存时将 `[[标题]]` 解析为显式 `Link` 对象，SQLite 内部绑定 `note_id`。
5. 详情页显示 Backlinks，按“提及 / 明确关系”分组。
6. 大规模（10k）下联想默认 Top N（建议 20）+ 分页加载。
7. 标题重名时，候选列表必须展示足够上下文供用户选择具体笔记；正文仍显示 `[[标题]]`，关系表保存确定的 `note_id`。

## 5.6 索引中心

职责：
- 管理四类索引：topic / nearby / logic_chain / free_link
- 维护索引项顺序与说明
- 维护主题索引笔记（索引区）

规则：
1. 索引笔记是“特殊原创笔记”，用于串联同主题原创笔记。
2. 原创笔记净新增每 15 条，触发一次“整理主题索引”提醒。
3. 索引笔记必须保存为 Markdown，同时结构化索引关系写入 SQLite。

## 5.7 图谱工作台

职责：
- 本地图、主题图、写作图
- 路径查找、冲突视图、过滤与聚焦

默认策略：
1. MVP 只显示当前目录下笔记之间的链接关系。
2. 默认从局部进入，不自动打开全局图。
3. 节点超阈值自动折叠与聚合。
4. 支持关系过滤、邻居分层展开、路径高亮。

## 5.8 写作中心

职责：
- 管理写作项目和写作篮
- 选择标签或原创笔记
- 生成写作提示词
- 生成提纲
- 导出 Markdown / DOCX

写作建议八步（引导流程）：
1. 选定写作主题
2. 明确读者与目标
3. 规划研究方案
4. 收集与筛选资料
5. 形成书摘笔记
6. 提炼原创笔记
7. 构建索引与写作篮
8. 生成并完善脚手架

MVP 范围：
1. 用户选择若干标签或原创笔记。
2. 系统基于选择内容生成提示词。
3. 系统基于提示词生成提纲。
4. MVP 暂不生成完整初稿正文。

---

## 6. AI 边界（强约束）

AI 可以：
- 分析资料
- 建议索引
- 建议连接
- 发现冲突
- 生成提示词与提纲

AI 不可以：
- 自动写并保存原创笔记正文
- 跳过用户确认直接产出最终成文
- 输出无证据链内容并默认采纳

---

## 7. 平台与国际化

1. v1 必须支持 Windows / macOS / Linux。
2. iOS / Android 为后续扩展，但架构需保持领域层与 UI 层解耦。
3. 前端文案必须资源化（i18n resource files），支持中文/英文切换。
4. 跨平台桌面技术路线建议采用 `Tauri + React/TypeScript + SQLite + CodeMirror 6`，兼顾本地目录体验、性能、包体与后续跨平台维护。

## 7.1 存储与索引规则

1. 目录树、索引笔记、所有笔记必须保存为 Markdown 文件。
2. 链接关系、标签关系、反向链接、搜索索引与图谱边写入 SQLite。
3. Markdown 是内容事实来源，SQLite 是关系索引与加速层。
4. SQLite 可重建，不得成为用户主内容唯一来源。

---

## 8. 验收导向（摘要）

v1 成功标准：
1. 用户能从资料形成书摘笔记。
2. 用户能从书摘笔记形成原创笔记。
3. 用户能通过索引与连接建立结构。
4. 用户能通过图谱发现可写路径。
5. 用户能让初稿从原创笔记中生长出来。
