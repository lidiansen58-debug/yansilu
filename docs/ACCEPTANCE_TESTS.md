# 研思录验收测试说明 v1.1

## 1. 文档目标

本文档定义 v1.1 的验收标准、关键流程与发布门槛，重点验证核心链路与目录模型。

---

## 2. 核心验收原则

v1.1 通过与否，以以下五点是否成立为准：

1. 用户能从资料形成书摘笔记
2. 用户能从书摘笔记形成原创笔记
3. 用户能围绕原创笔记建立索引与连接
4. 用户能通过图谱与索引进入写作篮
5. 系统能基于选择的标签或原创笔记生成提示词与提纲

---

## 4. 环境要求

- Windows 最新主流版本
- macOS 最新主流版本
- Linux 最新主流版本

## 5. P0 流程验收

## 5.0 流程 A0：目录初始化与子目录管理

目标：验证目录模型可用。

步骤：
1. 设置工作区根目录
2. 首次进入系统
3. 检查默认目录：随笔目录 / 书摘目录 / 原创目录
4. 在原创目录下新建子目录，最大笔记数保持默认 500
5. 通过“浏览...”选择本机保存目录
6. 隐藏一个子目录，再恢复显示
7. 删除一个子目录

通过标准：
- 默认目录存在且不可误删
- 子目录可新建/隐藏/删除
- 默认最大笔记数为 500，且可修改
- 新建目录支持系统目录选择器

失败条件：
- 默认目录缺失
- 子目录无法隐藏或删除
- 新建时没有上限配置能力
- 新建时只能手输路径，没有浏览选择入口

## 5.1 流程 A：从零创建书摘笔记

步骤：
1. 新建书摘笔记
2. 输入作者、年份、书名、出版社、页码
3. 输入摘录文字与转述
4. 保存

通过标准：
- 保存成功且字段完整持久化
- 页码和转述可回显

失败条件：
- 缺少页码或转述仍可无提示保存
- 元数据字段保存丢失

## 5.2 流程 B：从书摘笔记创建原创笔记

步骤：
1. 打开书摘笔记
2. 点击“新增原创笔记”
3. 填写标题、核心观点、为什么成立、边界/反例
4. 绑定引用并保存

通过标准：
- 原创笔记创建成功
- 与书摘笔记映射存在
- 在原创笔记中可查看来源引用
- 原书摘笔记保持原状态，不被自动归档或强制标记为已转换
- 原书摘笔记可链接到新原创笔记

失败条件：
- 允许空核心观点通过
- 来源链路断裂
- 从书摘创建原创后，书摘内容被覆盖或状态被强制改变

## 5.3 流程 C：原创性守卫拦截复制

通过标准：
- 复制大段摘录且相似度 `>= 80%` 时触发 blocked
- blocked 时禁止保存文件
- 提示可读、可操作

失败条件：
- 相似度 `>= 80%` 仍可保存 Markdown 文件

## 5.4 流程 D：创建主题索引

通过标准：
- 索引项可排序并持久化
- 可回跳到原创笔记
- 可被写作项目引用

## 5.5 流程 E：自由连接

通过标准：
- 连接双侧可见
- relation_type 保留
- free_link 要求 rationale

## 5.6 流程 F：查看当前目录链接图

通过标准：
- MVP 可显示当前目录下笔记之间的链接关系
- 不默认打开全局图
- 节点详情可回跳

## 5.7 流程 G：创建写作项目与写作篮

步骤补充：
- 尝试添加非原创目录内容到写作篮

通过标准：
- 仅原创目录中的原创笔记可加入写作篮
- 非原创目录内容被阻止并提示原因

## 5.8 流程 H：生成写作提示词与提纲

通过标准：
- 可选择标签或原创笔记
- 系统基于选择内容生成提示词
- 系统基于提示词生成提纲
- 不直接生成完整初稿正文

## 5.9 流程 I：原创笔记关联与反向链接（Obsidian 风格）

步骤：
1. 在原创笔记 A 中输入 `[[` 搜索原创笔记 B
2. 插入 `[[标题]]`
3. 设置 relation_type 与 rationale
4. 保存
5. 打开 B 查看 Backlinks

通过标准：
- 联想候选可用
- `[[...]]` 解析为 Link
- SQLite 内部保存确定的 `note_id`
- Backlinks 按“提及/明确关系”分组
- free_link 无 rationale 时阻断保存

补充场景：标题重名
1. 准备两条同标题原创笔记
2. 输入 `[[标题]]` 时从候选列表选择其中一条
3. 保存后检查 SQLite Link 指向被选择的 `note_id`

通过标准：
- 正文显示 `[[标题]]`
- 内部绑定的 `note_id` 不混淆

## 5.10 流程 J：随笔记录生命周期

步骤：
1. 新建随笔记录（文本/语音）
2. 保持未处理，检查“新建”标记
3. 将记录时间模拟超过 7 天
4. 查看默认列表与“显示隐藏项”
5. 从该随笔点击“编写原创笔记”并保存原创笔记
6. 从另一条随笔点击“转为书摘笔记”并保存书摘笔记

通过标准：
- 支持文本与语音
- 新建未处理有标记
- 超过 7 天默认隐藏
- 完成原创/书摘转换后原随笔自动归档

## 5.11 流程 K：15 条原创笔记索引整理提醒

步骤：
1. 在同一目录内连续新增原创笔记
2. 到第 15 条净新增时观察系统提醒
3. 点击提醒进入该目录索引区

通过标准：
- 第 15 条时触发一次提醒
- 可跳转到对应索引区
- 提醒可标记已处理

## 5.12 流程 L：原创笔记编辑器菜单与标签检索

步骤：
1. 打开原创笔记编辑器，确认顶部存在“编辑/插入/关联/标签”菜单与工具栏
2. 点击“关联笔记”按钮，验证等效触发 `[[` 联想面板
3. 选择一个目标原创笔记并插入链接后保存
4. 点击“标签”按钮，新增标签并保存
5. 在笔记正文中点击该标签链接
6. 查看标签结果页数据范围

通过标准：
- 编辑器提供可点击菜单与工具栏，不依赖纯 Markdown 手输
- “关联笔记”按钮行为等效 `[[`，且可插入结构化关联
- 标签可新增、可回显、可点击
- 点击标签后只展示“当前目录内”的原创笔记结果
- 标签结果页支持排序与回跳原笔记

失败条件：
- 缺少可点击菜单/按钮，仅能靠快捷键或手输
- “关联笔记”按钮不能触发联想
- 标签点击后跨目录混查
- 标签结果无法回跳原笔记

## 6. 模块级标准（摘要）

## 6.1 目录模块

必须通过：
- 可设置根目录
- 默认三类目录存在
- 子目录新建/隐藏/删除可用
- 主内容仍是 Markdown
- 新建目录支持浏览选择本机目录

## 6.2 随笔记录

必须通过：
- 文本/语音输入
- 7天隐藏
- 新建标记
- 转原创后自动归档

## 6.3 书摘笔记

必须通过：
- quote/paraphrase 分离
- 书摘元数据字段可编辑可保存
- 可直接新增原创笔记

## 6.4 原创笔记

必须通过：
- 结构化字段完整
- `[[wikilink]]` + Backlinks 可用
- 编辑器具备菜单栏与 Markdown 工具栏
- 提供“关联笔记”按钮（等效 `[[`）
- 提供“标签”按钮（等效 `#`）
- 标签点击后可查看当前目录内全部同标签原创笔记
- 原创性守卫生效

## 6.5 写作中心

必须通过：
- 仅原创目录内容可用于写作
- 可选择标签或原创笔记生成提示词
- 可基于提示词生成提纲
- 不直接生成完整初稿正文

## 6.6 i18n

必须通过：
- UI 文案来自资源文件（非硬编码）
- 中英切换后关键术语正确映射

---

## 7. 发布门槛

发布前必须满足：

1. 流程 A0、A-H、I、J、K、L 全通过
2. 无 P0 缺陷
3. AI 输出经 schema 校验
4. 主内容可导出
5. 导入具备预览、确认、ImportRecord
6. 至少完成一次端到端演示：空白目录 -> 书摘/原创 -> 链接 -> 提纲

---

## 8. P0 缺陷示例

出现以下任一项不得发布：
- AI 自动代写并保存原创笔记正文
- 主内容丢失且不可恢复
- 写作流程绕过原创笔记直接生成完整初稿
- 导入覆盖用户二次编辑且不可恢复
- 非原创目录内容可绕过限制进入写作流程
- 相似度 `>= 80%` 的原创笔记仍可保存文件
