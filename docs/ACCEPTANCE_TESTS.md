# 研思录验收测试说明 v1.3

更新日期：2026-06-03

## 1. 文档目标

本文件只定义当前 MVP 的发布门槛和关键验收流程。

它验证的是：

- 当前版本是否真实可用
- 当前范围是否与代码实现一致

它不再承担这些目标：

- 为未来连接器预写详细验收脚本
- 维护旧的导入历史/回滚模型

## 2. 当前验收原则

当前版本通过与否，以以下事实是否成立为准：

1. 用户能稳定使用本地 Vault 与目录树
2. 用户能创建、编辑、保存三类笔记
3. 用户能通过 `Obsidian 预览 -> 确认` 的方式把资料带入系统
4. 用户能导出 Markdown
5. 用户能基于永久笔记继续进入写作流程
6. 系统不会绕过原创性边界或静默破坏本地内容

## 3. 当前发布门槛

### 3.1 Gate A：浏览器原型 + 真实 API

以下能力必须通过：

1. 工作区初始化与切换
2. 目录 CRUD 与路径设置
3. 笔记 CRUD、保存反馈、草稿恢复
4. 索引、关联、标签、反向链接、图谱
5. Obsidian 导入预览与确认
6. Markdown 导出
7. 写作项目与脚手架草稿
8. 原创性守卫

### 3.2 Gate B：桌面可运行验证

以下能力是桌面 MVP 的额外门槛：

1. 原生目录选择器可用
2. 可在文件管理器中打开目录
3. 可定位当前 Markdown 文件
4. Windows 真机烟测通过
5. 桌面构建前检查与打包前检查可执行

### 3.3 当前导入导出阻塞范围

当前版本的导入导出验收，只以下列能力作为阻塞项：

1. Obsidian 导入
2. Markdown 导出
3. 导入预览与确认
4. 附件随导入/导出处理
5. 简化版导入边界清晰

以下来源即使仓库中出现实验性实现，也不作为当前发布阻塞项：

1. Notion
2. Readwise
3. Zotero
4. Evernote
5. Logseq
6. Roam
7. OneNote / Apple Notes

## 4. P0 必测流程

### 4.0 流程 A0：工作区初始化与切换

步骤：

1. 设置工作区根目录
2. 首次进入系统
3. 检查默认目录是否创建
4. 切换到另一个 Vault
5. 检查目录树、笔记、图谱与写作数据是否同步切换

通过标准：

1. 默认目录存在
2. `.yansilu` 结构已初始化
3. 切换 Vault 后不会残留上一工作区的数据

### 4.1 流程 A1：目录管理

步骤：

1. 新建目录
2. 设置目录保存路径
3. 重命名目录
4. 移动目录
5. 隐藏并恢复目录
6. 删除非默认目录

通过标准：

1. 目录元数据持久化正确
2. 修改路径后子目录与笔记映射保持一致
3. 默认目录不可误删

### 4.2 流程 A2：笔记创建、编辑与保存

步骤：

1. 在不同笔记目录下创建笔记
2. 编辑正文
3. 观察脏状态提示
4. 手动保存
5. 刷新后检查草稿恢复
6. 移动或删除一条笔记

通过标准：

1. 笔记类型与目录一致
2. 保存后内容真实写入 Markdown
3. 有可理解的脏状态与保存反馈
4. 草稿恢复有效
5. 移动和删除后目录树与文件系统一致

### 4.3 流程 A3：关联与标签

步骤：

1. 在正文输入 `[[`
2. 选择另一条笔记
3. 保存
4. 打开目标笔记查看反向链接
5. 点击 `#tag`

通过标准：

1. `[[...]]` 候选联想可用
2. 正文保持 Markdown 可读
3. 关系能落到真实目标笔记
4. 反向链接可见
5. 标签点击结果可解释

### 4.4 流程 A4：图谱

步骤：

1. 打开图谱视图
2. 查看当前目录范围关系
3. 切换不同图谱视角
4. 点击节点查看详情
5. 从节点回到笔记

通过标准：

1. 图谱默认展示当前范围
2. 不默认渲染无界全量图
3. 节点详情可回到真实笔记

### 4.5 流程 A5：Obsidian 导入预览与确认

步骤：

1. 选择 Obsidian 输入源
2. 执行导入预览
3. 检查候选分类、警告与阻塞原因
4. 排除部分候选后确认导入

通过标准：

1. 所有导入都先预览后确认
2. 预览结果能区分可导入、警告、阻塞、排除
3. 确认结果能展示创建结果与跳过原因
4. 确认后能继续进入后续整理或写作链路
5. 嵌入资源会复制到 vault 内可用路径

失败条件：

1. 没有预览就直接写入
2. 风险候选没有理由说明
3. 导入后的资源链接不可用或文件未复制

### 4.6 流程 A6：Markdown 导出

步骤：

1. 选择一组可导出的笔记
2. 执行 Markdown 导出
3. 打开导出目录检查结果

通过标准：

1. 导出成功
2. 导出结构与内容可读
3. 用户能定位输出目录

### 4.7 流程 A7：写作项目与脚手架

步骤：

1. 从永久笔记组织写作范围
2. 创建写作项目
3. 生成脚手架草稿
4. 查看章节、证据映射、缺口与反方

通过标准：

1. 写作入口来自已有永久笔记
2. 写作项目创建成功
3. 脚手架结构完整且可追溯
4. 结果是结构骨架，而不是完整代写正文

### 4.8 流程 A8：原创性守卫

步骤：

1. 在永久笔记中粘贴大段来源原文
2. 尝试保存
3. 在导入预览中准备高风险永久笔记候选
4. 检查阻断或警告结果

通过标准：

1. 永久笔记保存会触发守卫
2. 高相似度内容会被阻断或明确警告
3. 导入流程中的永久笔记候选也受同类检查

## 5. 桌面额外烟测

以下流程在桌面版上必须补测：

1. 打开原生目录选择器并返回路径
2. 从目录菜单打开该目录在文件管理器中的位置
3. 从笔记菜单定位当前 Markdown 文件
4. 验证中文路径、空格路径可用
5. 验证弹窗不会被主窗口遮挡

## 6. 当前不作发布阻塞项的内容

以下能力可以继续推进，但不作为当前发布阻塞项：

1. 语音输入
2. 完整双语 i18n
3. 默认全局图谱
4. 语义搜索与连接建议
5. 更重型的外部连接器能力
6. 完整桌面安装器体验

## 7. Future Work Note

未来如果重新引入 Notion / Readwise / Zotero 等连接器，应为它们单独补充正式验收流程。

在那之前，不要把下列能力重新写回当前阻塞门槛：

1. 导入历史持久化
2. 导入回滚
3. Markdown 导入
4. 多连接器并列承诺
