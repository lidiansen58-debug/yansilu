# 研思录验收测试说明 v1.2

## 1. 文档目标

本文档用于定义当前 MVP 的发布门槛与关键验收流程。

它验证的是“当前版本是否真的可用”，不是“理想态是否已经全部做完”。

---

## 2. 验收原则

当前版本通过与否，以以下五点是否成立为准：

1. 用户能稳定使用本地工作区与目录树。
2. 用户能创建、编辑、保存并关联三类笔记。
3. 用户能把资料通过导入预览/确认的方式带入系统。
4. 用户能基于永久笔记建立写作项目并生成脚手架草稿。
5. 系统不会绕过原创性边界或静默破坏本地内容。

---

## 3. 发布门槛分层

## 3.1 Gate A：浏览器原型 + 真实 API

以下能力必须通过：

1. 工作区初始化与切换。
2. 目录 CRUD 与路径设置。
3. 笔记 CRUD、保存反馈、草稿恢复。
4. 索引、关联、标签、反向链接、图谱。
5. 导入预览、确认、历史、回滚。
6. Markdown 导出。
7. 写作项目与脚手架草稿。
8. 原创性守卫。

## 3.2 Gate B：桌面可运行验证

以下能力是“desktop-runnable MVP”额外门槛：

1. 原生目录选择器可用。
2. 可在文件管理器中打开目录。
3. 可定位当前 Markdown 文件。
4. Windows 真机烟测通过。
5. 桌面构建前检查与打包前检查可执行。

---

## 3.3 当前导入导出阻塞范围

当前版本的导入导出验收，只以下列能力作为阻塞项：

1. Markdown 导入。
2. Obsidian 导入。
3. Markdown 导出。
4. 导入预览、确认、记录、回滚。
5. 附件随导入/导出处理。

以下来源即使代码中已有实验性实现，也暂不作为当前发布阻塞项：

1. Notion。
2. Readwise。
3. Zotero。
4. Evernote。
5. Logseq。
6. Roam。
7. OneNote / Apple Notes。

原因不是这些来源不重要，而是它们属于路线图后续阶段，不应和当前 `Phase 1` 的开放格式底座混为一谈。

## 4. P0 必测流程

## 4.0 流程 A0：工作区初始化与切换

步骤：

1. 设置工作区根目录。
2. 首次进入系统。
3. 检查默认目录是否创建。
4. 切换到另一套 Vault。
5. 检查目录树、笔记、图谱与写作数据是否同步切换。

通过标准：

1. 默认目录存在：`fleeting / literature / original`。
2. `.yansilu` 结构与 SQLite 文件已初始化。
3. 切换 Vault 后，旧工作区数据不会残留在当前界面。

失败条件：

1. 默认目录缺失。
2. 初始化后无法保存后续笔记。
3. 切换 Vault 后仍显示上一工作区的数据。

## 4.1 流程 A1：目录管理

步骤：

1. 在永久笔记目录下新建子目录。
2. 设置目录保存路径。
3. 重命名目录。
4. 拖拽或移动目录到另一个父目录。
5. 隐藏目录并恢复显示。
6. 删除非默认目录。

通过标准：

1. 目录的 `name`、`parentId`、`fsPath`、`hidden` 状态都正确持久化。
2. 修改目录路径后，子目录与笔记路径映射保持一致。
3. 默认目录不可误删。

失败条件：

1. 目录移动后笔记丢失或路径断裂。
2. 设置路径后出现目录树与真实文件位置不一致。
3. 默认目录被删除。

## 4.2 流程 A2：笔记创建、编辑与保存

步骤：

1. 分别在随笔、书摘、永久笔记目录中创建笔记。
2. 打开一条笔记进行编辑。
3. 观察脏状态提示。
4. 手动保存。
5. 再次修改并刷新，检查草稿恢复。
6. 移动或删除一条笔记。
7. 对来自书摘转化或建议回填的永久笔记，检查保存前的作者确认动作。

通过标准：

1. 笔记类型与所属根目录一致。
2. 保存后内容真实写入 Markdown。
3. 存在可理解的脏状态与保存反馈。
4. 草稿恢复有效。
5. 移动和删除后目录树与实际文件一致。
6. 永久笔记不会把外部摘录或 AI 建议静默保存成“已拥有的观点”，而是要求用户做明确确认。

失败条件：

1. 保存只更新 UI，不更新本地文件。
2. 草稿恢复覆盖了已正式保存的内容。
3. 移动或删除后 SQLite 与文件系统状态不一致。
4. 永久笔记保存流程把接受建议等同于拥有判断，没有任何作者确认或改写提醒。

## 4.3 流程 A3：永久笔记关联与标签

步骤：

1. 在永久笔记 A 中输入 `[[`。
2. 选择永久笔记 B 作为目标。
3. 保存笔记。
4. 打开 B，查看反向链接。
5. 在正文中点击一个 `#标签`。

通过标准：

1. `[[...]]` 候选联想可用。
2. 正文仍为 Markdown 可读形式。
3. 内部关系绑定到确定的 `note_id`。
4. 反向链接可见。
5. 标签点击结果默认覆盖当前根目录范围，而不局限于当前已展开子目录。

失败条件：

1. 标题重名时无法区分目标笔记。
2. 保存后只保留纯文本，不生成结构化关系。
3. 标签点击只能检索当前已加载的一小部分数据。

## 4.4 流程 A4：问题导向图谱

步骤：

1. 选择一个存在若干关联笔记的目录。
2. 打开图谱视图。
3. 确认默认进入本地图。
4. 切换到主题图或写作图。
5. 检查支持关系、冲突关系或桥接缺口提示。
6. 点击节点查看详情。
7. 从节点回跳到笔记。

通过标准：

1. 图谱默认展示当前目录范围的关系。
2. 不默认打开全局全量图。
3. 图谱至少支持本地图、主题图、写作图中的问题导向视角。
4. 用户能够在图谱中区分支持关系、冲突关系，或看到明确的桥接缺口提示。
5. 节点详情可回跳到真实笔记。

失败条件：

1. 图谱默认渲染全局大图导致不可读。
2. 图谱只能展示“有连接”，却无法表达支持、冲突或结构缺口。
3. 图谱中的节点无法定位回笔记。

## 4.5 流程 A5：导入预览、确认、历史与回滚

步骤：

1. 选择 Markdown 或 Obsidian 输入源。
2. 执行导入预览。
3. 检查候选项分类、警告与阻断原因。
4. 排除部分候选后确认导入。
5. 打开导入历史。
6. 对一条可回滚记录执行回滚。

通过标准：

1. 所有导入都先预览后确认。
2. 预览结果能区分可导入、警告、阻断、排除。
3. 确认结果能展示创建结果与跳过原因。
4. 导入历史可读。
5. 回滚不会静默覆盖用户后来手动改过的文件。

失败条件：

1. 没有预览就直接落库。
2. 风险候选没有理由说明。
3. 回滚误删用户后续修改内容。

## 4.6 流程 A6：Markdown 导出

步骤：

1. 选择一组可导出的笔记内容。
2. 执行 Markdown 导出。
3. 打开导出目录检查结果。

通过标准：

1. 导出成功。
2. 导出结构与内容可读。
3. 导出后用户能定位输出目录。

失败条件：

1. 导出只返回成功提示，但没有真实文件。
2. 导出内容缺失或结构不可用。

## 4.6.1 后续连接器验收占位（当前非阻塞）

以下连接器在进入对应阶段时，需要补充各自的正式验收流程，但当前版本先不以此阻塞发布：

1. Notion：验证官方导出 zip 的页面层级、数据库与附件处理。
2. Readwise：验证 highlights、notes、tags 与来源元数据映射。
3. Zotero：验证引用定位信息、条目元数据与批注映射。
4. Evernote / Logseq / Roam：验证内容保留率与结构降级策略。

进入正式验收前，所有连接器都仍需满足统一导入契约：

1. 先预览，后确认。
2. 有导入记录。
3. 有跳过原因。
4. 有可解释的风险提示。
5. 有回滚边界。

## 4.7 流程 A7：写作项目与脚手架草稿

步骤：

1. 从主题索引或若干永久笔记组织写作篮。
2. 创建写作项目。
3. 生成脚手架草稿。
4. 查看脚手架分节/段落块、证据映射、缺口、反方与待补问题。
5. 导出或复制脚手架 Markdown。

通过标准：

1. 主题索引可以作为写作入口，但写作篮中的实际材料必须解析为永久笔记。
2. 写作项目创建成功。
3. 脚手架草稿结构完整且可追溯到证据笔记。
4. 脚手架显式标出缺口、反方或待补问题，而不是把冲突抹平。
5. 输出结果是“结构骨架”，不是完整终稿。
6. 写作入口默认来自已有永久笔记/写作篮，而不是空白 prompt 代写入口。

失败条件：

1. 允许书摘笔记或原始摘录直接进入写作篮。
2. 脚手架没有证据映射。
3. 脚手架隐藏结构缺口与反方。
4. 系统默认返回完整代写正文。
5. 写作中心把“让 AI 从零写完”作为主路径，而不是从既有思想结构出发。

## 4.8 流程 A8：原创性守卫

步骤：

1. 在永久笔记中粘贴大段来源原文。
2. 尝试保存。
3. 在导入预览中准备高风险永久笔记候选。
4. 检查阻断或警告结果。

通过标准：

1. 永久笔记保存会触发守卫。
2. 高相似度内容会被阻断或明确警告。
3. 导入流程中的永久笔记候选也受到同类检查。

失败条件：

1. 守卫可被保存流程绕过。
2. 高风险内容没有任何提示或说明。

---

## 5. 桌面额外烟测

以下流程在桌面版本上必须补测：

1. 打开原生目录选择器并返回路径。
2. 从目录菜单打开该目录在文件管理器中的位置。
3. 从笔记菜单定位当前 Markdown 文件。
4. 验证中文路径、空格路径可用。
5. 验证弹窗不会被主窗口遮挡。

---

## 6. 当前版本不作为阻塞项的内容

以下能力可以继续推进，但当前不作为发布阻塞项：

1. 语音输入。
2. 完整双语 i18n。
3. 默认全局图谱。
4. 语义搜索与连接建议。
5. 更重型的外部连接器能力。
6. 完整桌面安装器体验。

## 7. 下一轮界面优化方向（非当前阻塞）

以下要求不阻塞当前 API/Web 可运行验收，但应作为下一轮产品表达优化的明确方向：

1. 首页主指标优先展示“形成了多少永久笔记”，而不是“导入了多少资料”。
2. 文献笔记与永久笔记在首页、列表与搜索中保持严格分层，不混成同类结果。
3. 搜索结果默认优先返回可复用观点，再返回转述和原始摘录。
4. 首页和主按钮文案不把产品包装成“AI 帮你写完”的外部神力工具。
