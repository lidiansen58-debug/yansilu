# 研思录 V1.1 后续任务重规划

## 1. 命名决策

从本阶段开始，产品内外部统一使用 `观点提纯`，不再把 `思想提纯器` 作为主命名。

原因：

1. `观点提纯` 更像用户能理解的具体动作。
2. 它能直接对应当前产品里的 `一句话判断`、`三句话压缩`、`确认观点`、`进入写作准备`。
3. 它比 `思想提纯器` 更少哲学感，降低新用户理解成本。
4. 它仍然保留原有产品内核：不是资料总结，而是把材料变成用户愿意承担的个人判断。

统一口径：

`研思录帮助用户把材料提炼成自己的观点，再把观点组织成可写作的结构。`

---

## 2. 当前完成状态

截至 `2026-05-17`，V1.1 已完成或基本完成：

1. 永久笔记观点字段：`thesis`、`threeLineSummary`、`distillationStatus`。
2. 永久笔记右侧观点提纯面板：保存观点、确认观点。
3. 观点提纯队列：待提纯条目、队列入口、笔记定位。
4. 用户确认边界：`confirmed` 必须来自显式用户动作。
5. 写作项目意图字段：`intent`、`desiredReaderTakeaway`。
6. 写作 scaffold 前检查：非阻断式 readiness/preflight。
7. AI Inbox 与候选态基础：AI 建议默认 review-only，不直接改写或确认用户判断。

## 3. 第一阶段：V1.1 Alpha 收口

目标：让 `观点提纯` 在首页、笔记、队列、写作出口中形成一致感知。
状态：已于 `2026-05-17` 收口完成。验收记录见 `docs/V1_1_ALPHA_ACCEPTANCE_RECORD_2026_05_17.md`。

### A1. 术语统一：从思想提纯迁移到观点提纯

范围：

1. 产品导航、按钮、面板标题、空态、提示文案。
2. 官网/产品说明中的主表达。
3. 当前活跃规划文档中的阶段名称与验收口径。
4. API 文档中的用户可见描述。

不做：

1. 不批量改写历史研究文档中的原始论述。
2. 不改数据库字段名，例如 `distillationStatus` 可继续作为工程字段。
3. 不把 `观点提纯` 解释成 AI 摘要或自动总结。

验收：

1. 用户界面主路径不再出现 `思想提纯器` 作为功能名。
2. 产品文案优先使用 `观点提纯`、`写一句判断`、`压缩成三句话`、`确认观点`。
3. 文档中明确 `思想提纯器` 是历史/旧称，后续主称为 `观点提纯`。

### A2. 首页价值反馈重排

目标：让首页首先展示观点形成进度，而不是资料积累量。

任务：

1. 展示待提纯永久笔记数。
2. 展示已确认观点数。
3. 展示可进入写作的观点数。
4. 展示缺口提醒，例如缺一句话判断、缺三句话压缩、缺反方/边界。
5. 首页主行动指向 `继续观点提纯` 或 `进入写作准备`。

### A3. 提纯队列 polish

目标：让队列成为轻量工作入口，而不是只读列表。

任务：

1. 增加队列状态筛选：待一句话判断、待三句话压缩、待确认。
2. 点击队列项后定位到对应永久笔记的提纯面板。
3. 保存或确认后队列即时刷新。
4. 优化空态：没有待提纯时，引导进入写作准备。

### A4. V1.1 Alpha 验收文档

任务：

1. 更新已实现 API 与 UI 范围。
2. 记录全量测试结果。
3. 标记 Alpha 已完成、不进入 Alpha 的能力、进入 Beta 的候选能力。

---

## 4. 第二阶段：观点质量检查层

目标：让用户知道观点哪里还不清楚，而不是只保存字段。

任务：

1. 检查一句话判断是否为空、过短、过长、像标题。
2. 检查三句话压缩是否恰好三条、是否重复、是否缺少用途。
3. 检查反方/边界是否缺失。
4. 将质量提示接入提纯面板与写作 scaffold preflight。
5. 保持非阻断：提示可以推动修改，但不强制拦截普通保存。

---

## 5. 第三阶段：AI 观点候选

目标：让 AI 成为候选生成器，而不是观点作者。

任务：

1. 永久笔记 AI 生成候选 `thesis`。
2. 永久笔记 AI 生成候选 `threeLineSummary`。
3. AI 输出必须进入 AI Inbox 或候选卡片。
4. 用户可采纳为 `draft`，但不能直接变成 `confirmed`。
5. 候选需带 rationale/source scope，说明依据来自哪里。

---

## 6. 第四阶段：主题与中心问题

目标：把单条观点提纯推进到主题级组织。

任务：

1. IndexCard 中心问题 UI 产品化。
2. 主题页展示中心问题、主题观点、相关永久笔记、写作入口。
3. 写作项目优先从主题/中心问题进入。
4. 为后续张力图和主题工作区保留结构。

---

## 7. 暂缓任务

以下任务不进入 V1.1 Alpha：

1. 独立三栏式观点提纯工作区。
2. 全库 AI 扫描。
3. 自动生成完整文章。
4. 复杂张力图。
5. 团队协作。

---

## 8. 下一刀建议

下一步优先做：

`A1. 术语统一：从思想提纯迁移到观点提纯`

理由：

1. 用户刚进入产品时，命名决定第一理解。
2. 当前功能已经基本具备，先统一表达能减少认知噪音。
3. 它会影响首页、提纯面板、队列、写作入口的后续文案设计。
