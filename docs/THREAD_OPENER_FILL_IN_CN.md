# Thread Opener Fill-In CN

把尖括号里的内容替换掉，就可以直接作为新 thread 开场词使用。

## 1. Bugfix

```text
工作目录：<工作目录>
分支：fix/<分支名>
目标：修复 <一句话 bug>

先不要跑测试。
先检查相关文件并确认复现路径。
只处理当前 bug。
只改：<模块/文件范围>
修改后只跑最小相关验证：<命令>
达到可 review 状态就停。
```

## 2. Feature

```text
工作目录：<工作目录>
分支：feat/<分支名>
目标：<一句话功能目标>

先不要跑测试。
只处理当前功能目标。
只改：<模块/文件范围>
代码组织要求：
- 不要把新的业务逻辑继续堆进 prototype 主文件
- 如果引入新职责，优先拆出一个直接相关的小模块
- prototype 主文件只负责装配、接线和模块组合
修改后只跑最小相关验证：<命令>
达到可 review 状态就停。
```

## 3. 涉及中文文本时追加

```text
编码要求：
- 所有新建或修改的文本文件按 UTF-8 处理
- 如果脚本会写文件，必须显式指定 UTF-8
- 如果终端显示异常，先用显式 UTF-8 方式读取确认，不直接判断文件损坏
```

## 4. 最小替换原则

每次只需要先填这几个字段：

- `<工作目录>`
- `<分支名>`
- `<一句话目标>`
- `<模块/文件范围>`
- `<命令>`

不要在开场词里提前塞太多背景、历史、远期计划。
