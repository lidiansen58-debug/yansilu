# 研思录 UI 原型方案 P0 v1.2

## 1. 原型目标

本原型用于让设计师和研发快速理解 v1 核心交互，不追求视觉细节，优先验证：

1. 5 分钟最小闭环可跑通
2. 复杂度按层解锁，不一次性压给用户
3. 书摘笔记 -> 原创笔记转化足够顺
4. 原创笔记之间的关系网络，而不是孤立卡片，能够直观可见
5. 10,000 原创笔记下仍可操作

---

## 2. 信息架构（分层）

## L0 入门层（默认可见）

1. 原创笔记工作台
2. 新建原创笔记
3. 素材入口（折叠收纳随笔记录、文献笔记和导入候选）

## L1 成型层（形成首条原创后解锁）

1. 原创笔记提纯面板
2. 从素材记录原创

## L2 结构层（完成首条原创后解锁）

1. 当前节点关系面板
2. 索引中心
3. 图谱工作台

## L3 写作层（完成首个索引后解锁）

1. 写作中心

---

## 3. 页面与关键交互

## 3.1 思想工作台首页

包含：
1. 原创笔记默认入口
2. 子卡片盒管理（新建、隐藏、删除）
3. 新建卡片盒时设置“最大卡片数量”（默认 500）
4. 首页主指标区：本周新增原创笔记、本周接入网络的原创笔记、本周新建主题索引、本周可进入写作的主题
5. onboarding 进度条（0/5 -> 5/5）
6. 单一主行动按钮：新建原创笔记

关键反馈：
- 首页默认强调“形成了多少原创笔记”，而不是“导入了多少资料”。
- 随笔和文献不再作为首页一级主对象，而是作为“素材入口”收纳。
- 导入量、来源量、连接器量只作为二级信息，不占据主 KPI。
- 不把“你拥有多少资料”包装成进展，避免把收藏感误认为思考成果。
- 不把“AI 帮你写完”作为首页主按钮或主文案，避免把产品包装成外部神力代理。
- 未解锁模块显示“为什么未解锁 + 如何解锁”。
- 每次完成动作后显示“你获得了什么价值”。

## 3.2 随笔记录

包含：
1. 文本输入 + 语音输入
2. 新建标记（未处理）
3. 超过 7 天内容默认隐藏（可筛选显示）
4. “编写原创笔记”快捷入口

关键流程：
- 随笔 -> 原创笔记保存成功 -> 原随笔自动归档。

## 3.3 书摘笔记工坊

包含：
1. 摘录或材料
2. 我的转述
3. 它可能支持的判断
4. 来源详情折叠区：作者/年份/标题/容器信息/页码或定位/版本/译者编者/DOI、ISBN、arXiv、URL 或 PDF 路径
5. 主按钮：从转述记录原创

关键反馈：
- 不设计单纯“收藏夹”，书摘默认进入待转述队列，等待完成转述、定位与后续提炼。
- 不突出累计摘录数，优先显示“还需转述”和“可提炼为原创”的进度信号。
- 文献界面默认不展示完整引用表单，避免把用户拖回资料管理。

## 3.4 原创笔记编辑器（重点）

包含：
1. 结构化字段：核心观点、为什么成立、边界/反例
2. 来源绑定与书摘引用管理
3. originality guard 反馈（pass/warning/blocked）
4. `[[wikilink]]` 输入与候选联想
5. 关系面板（relation_type + rationale）
6. Backlinks 面板（提及/明确关系）
7. Markdown 工具栏（可点按钮，不只键盘输入）
8. 标签面板（标签候选、标签管理、标签结果列表）
9. 编辑器小精灵提示（可随时关闭，提示当前任务、下一步推荐，后续预留接入大模型）

关键约束：
1. 不提供“AI 一键生成原创正文”。
2. `blocked` 不能保存 active。
3. `free_link` 缺少 rationale 时阻断提交。
4. 搜索与候选面板默认优先显示观点型原创笔记，而不是书摘摘录。
5. 原创性检测只对原创笔记可见，并以浮窗方式提示，不占用编辑器底部空间。
6. 标题按钮改为 1-6 级标题下拉框；`[[关联笔记]]` 与 `#标签` 放在工具栏最右侧，和普通格式按钮分组。
7. 编辑器底部不再承载长期状态解释，改为轻量浮动状态提示与小精灵提示。

慢思考约束：
1. 原创笔记编辑器默认采用低干扰布局，弱化任务感和冲刺感。
2. 提供“专注提炼模式”，进入后收起左侧导航与回链侧栏，只保留当前笔记、必要按钮与最少提示。
3. 状态文案优先显示“提炼中 / 成形中 / 已串联”等生长信号，不突出速度、频次或排行榜式提示。
4. 非必要提醒默认不打断写作，只有在原创性、证据链或结构完整性受影响时才给出强反馈。

快捷键区（原型必须展示）：
1. 写作：`Ctrl+B`、`Ctrl+I`、`Ctrl+Shift+H`、`Ctrl+\``、`Ctrl+/`、`Ctrl+E`、`Ctrl+R`
2. 结构：`Ctrl+1...6`、`Ctrl+L`、`Alt+Q`
3. 检索：`Ctrl+F`、`Ctrl+Shift+F`、`F3`、`Shift+F3`、`Ctrl+Shift+O`、`Ctrl+Shift+G`
4. Mac 提示：`Ctrl -> Cmd`，`Alt -> Option`

工具栏按钮区（原型必须展示）：
1. 加粗、斜体、高亮、行内代码、公式、引用、分割线
2. 标题下拉框（1-6 级标题）
3. `[[关联笔记]]` 按钮（点击即触发候选面板）
4. `#标签` 按钮（点击即触发标签候选面板）
5. 保存按钮

标签交互（原型必须展示）：
1. 点击标签后，进入“该标签结果页”
2. 结果范围默认限制为“当前卡片盒 + 原创笔记”
3. 结果页支持按更新时间排序与子卡片盒筛选

搜索优先级（原型必须展示）：
1. 原创笔记标题/核心观点优先于书摘摘录
2. 书摘转述优先于原始摘录
3. 当同主题同时存在原创笔记与书摘笔记时，原创笔记作为主结果，书摘作为证据副结果
4. 搜索结果摘要默认先显示观点句或转述，不默认用大段摘录占据首屏

## 3.5 索引中心

包含：
1. 四类索引创建与维护：主题索引、邻近索引、序列索引、自由连接索引
2. 每个卡片盒独立索引区
3. 主题索引笔记（特殊原创笔记）
4. 索引项排序、理由补充、回跳原笔记
5. 索引之间的交叉引用与切换入口

新增提醒：
- 原创笔记净新增每 15 条，触发“整理主题索引”提醒。

关键反馈：
- 索引不是标签墙，而是组织关系与生成结构的工作区。
- 用户应能看见一个索引为何容纳这些观点，而不是只看见一个名字。

## 3.6 图谱工作台

包含：
1. 本地图/主题图/写作图切换
2. 支持/冲突/桥接缺口过滤
3. 关系过滤、标签过滤、路径高亮
4. 节点详情侧栏与回跳

图上必须能回答的问题：
1. 哪些观点正在形成互相支持的论证团簇
2. 哪些观点之间存在冲突、张力或反例
3. 哪些主题之间缺少桥接节点，导致过渡断裂或难以进入写作
4. 当前写作项目还缺哪一类过渡性观点或证据节点

10k 交互策略：
1. 默认局部图（不默认全局图）
2. 节点阈值折叠与聚类
3. 邻居分层展开（1~3 hops）
4. 联想和搜索 Top N + 分页

## 3.7 写作中心

包含：
1. 写作项目管理
2. 写作篮管理
3. 仅允许“原创卡片盒”笔记进入写作篮
4. 脚手架生成与证据映射
5. 主题就绪提示（哪些主题已可进入写作）

写作引导（八步）：
1. 选主题
2. 设目标
3. 规划研究
4. 收集资料
5. 形成书摘
6. 提炼原创
7. 先确认可写主题，再组写作篮
8. 出脚手架

默认入口约束：
1. 写作中心的第一入口应是“从已有原创笔记开始”，不是“让 AI 从空白开始写”。
2. 主按钮文案优先使用“组织观点”“生成脚手架”“检查证据缺口”，不使用“AI 帮我写文章”。

---

## 4. 5 分钟首轮脚本（演示）

1. 新建一条原创笔记，写下一个自己的判断
2. 从一条随笔或文献材料记录第二条原创笔记
3. 为两条原创笔记建立带理由的关联
4. 建一个主题索引并加入 2~3 条原创笔记
5. 进入写作中心生成首份脚手架

---

## 5. 设计交付要求（给设计师）

每页提供：
1. 正常态、空态、错误态、加载态
2. onboarding 前/中/后状态
3. 未解锁/已解锁对照
4. 中文与英文双语文案占位（走 i18n key）
