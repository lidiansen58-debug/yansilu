# 论文工作台低保真页面清单与状态文案 V1

## 1. 文档用途

这份文档服务两个角色：

1. 设计师：明确需要出哪些页面、每页最少包含什么。
2. 前端原型：明确空态、加载态、风险态、成功态该怎么写。

它默认建立在以下文档基础上：

1. [NOTEBOOKLM_ASSISTED_PAPER_WORKFLOW_V1.md](/E:/Projects/Thinking%20in%20Notes/yansilu/docs/NOTEBOOKLM_ASSISTED_PAPER_WORKFLOW_V1.md)
2. [PAPER_WORKSPACE_UI_INTERACTION_V1.md](/E:/Projects/Thinking%20in%20Notes/yansilu/docs/PAPER_WORKSPACE_UI_INTERACTION_V1.md)

## 2. 低保真页面清单

这套工作流建议至少拆成 7 个低保真页面/状态组。

### Page 1：入口页

目标：

1. 让用户理解这不是普通导入器。
2. 让用户知道自己会完成“从论文到原创笔记”的转换。

最少包含：

1. 页面标题：`论文工作台`
2. 一句话说明：`把论文阅读推进为你自己的原创判断`
3. 两个入口按钮：
   - `新建论文来源`
   - `继续上次加工`
4. 最近论文列表
5. 当前阶段标签：`来源 / 转述 / 原创 / 关联`

### Page 2：论文来源建立页

目标：

1. 创建 Source。
2. 绑定 PDF / DOI / arXiv / URL。
3. 为后续 NotebookLM 内容提供归属。

最少包含：

1. 标题、作者、年份
2. DOI / arXiv / URL
3. PDF 上传或绑定
4. 研究主题标签
5. 次级入口：
   - `添加 NotebookLM summary`
   - `添加 NotebookLM Q&A`
   - `添加 NotebookLM notes`

### Page 3：NotebookLM 内容导入页

目标：

1. 接收 NotebookLM 文本。
2. 让用户知道这些内容仍然只是“外部整理结果”。

最少包含：

1. 来源类型切换：
   - Summary
   - Q&A
   - Study Guide
   - Notes
2. 大文本粘贴区
3. `解析为候选` 按钮
4. 风险提醒区：
   - `这些内容不会直接保存为原创笔记`

### Page 4：候选拆分与筛选页

目标：

1. 把长文本拆成小颗粒候选。
2. 让用户筛掉无价值候选。

最少包含：

1. 候选卡片列表
2. 类型标签：
   - 论点
   - 方法
   - 结果
   - 局限
   - 问题
3. 每张卡片的操作：
   - `加入转述区`
   - `跳过`
   - `查看原文`
4. 顶部统计：
   - 总候选数
   - 已选数
   - 已跳过数

### Page 5：转述加工页

目标：

1. 把外部候选推进为用户自己的文献笔记。
2. 强制发生“我的理解”。

最少包含：

1. 当前候选原文
2. 我的转述输入区
3. 引导问题 chips
4. 立场按钮：
   - `我同意`
   - `我保留`
   - `我不同意`
5. 与当前研究问题的关系输入
6. `保存为文献笔记` 按钮
7. `生成原创候选` 按钮

### Page 6：原创笔记候选页

目标：

1. 呈现原创判断骨架。
2. 不让它看起来像已经自动完成。

最少包含：

1. 候选标识：`原创笔记候选，尚未定稿`
2. 字段：
   - 核心判断
   - 为什么成立
   - 边界/反例
   - 依赖证据
3. 风险提醒：
   - 是否过于贴近来源文本
   - 是否缺少边界
   - 是否缺少证据
4. 操作：
   - `保存为原创笔记`
   - `继续修改`
   - `暂存候选`

### Page 7：保存成功与后续行动页

目标：

1. 让用户立即进入“关联、索引、写作”的下一步。
2. 不让流程停在“保存成功”。

最少包含：

1. 成功提示
2. 新建原创笔记摘要
3. 下一步建议：
   - `去找可连接的旧笔记`
   - `加入主题索引`
   - `放入写作项目`
   - `继续加工下一条候选`

## 3. 页面共用状态

每个页面建议至少设计这 5 类状态：

1. 空态
2. 加载态
3. 正常态
4. 风险态
5. 成功态

## 4. 状态文案表

### 4.1 入口页

空态：

- 标题：`还没有开始加工论文`
- 说明：`先建立一篇论文来源，再把 NotebookLM 帮你整理过的内容推进成你自己的判断。`
- 主按钮：`新建论文来源`

继续加工态：

- 标题：`继续你上次的思考加工`
- 说明：`上次停在转述阶段，离原创笔记只差一步。`
- 主按钮：`继续加工`

### 4.2 来源建立页

空态：

- 标题：`先给这篇论文一个明确来源`
- 说明：`后续所有文献笔记和原创笔记都会围绕这个来源生长。`
- 主按钮：`保存论文来源`

错误态：

- 标题：`还缺少关键来源信息`
- 说明：`至少补充标题，以及 DOI、URL、arXiv 或 PDF 其中一项。`
- 主按钮：`继续补充`

### 4.3 NotebookLM 内容导入页

空态：

- 标题：`粘贴 NotebookLM 帮你整理的内容`
- 说明：`可以先从 summary、Q&A 或 notes 开始。系统会先拆成候选，不会直接写成原创笔记。`
- 主按钮：`解析为候选`

加载态：

- 标题：`正在拆分候选`
- 说明：`我们在把长摘要切成更适合加工的小颗粒。`

风险态：

- 标题：`这些内容还只是外部整理结果`
- 说明：`你稍后需要补上自己的转述和判断，系统才会帮助生成原创笔记候选。`

### 4.4 候选拆分页

空态：

- 标题：`还没有可加工候选`
- 说明：`试试换一段更具体的 summary、Q&A 或 notes。`
- 主按钮：`返回补充内容`

正常态：

- 标题：`先挑出真正值得加工的部分`
- 说明：`不是每条摘要都值得留下。先选那些会改变你判断的内容。`

成功态：

- 标题：`已选出待加工候选`
- 说明：`下一步，把这些候选转成你的文献转述。`
- 主按钮：`进入转述区`

### 4.5 转述加工页

空态：

- 标题：`把它变成你的理解`
- 说明：`如果这里只是重写摘要，还不算完成转述。试着回答：它改变了你对什么问题的看法？`

输入中状态：

- 标题：`正在形成文献转述`
- 说明：`先写你怎么理解，再写它和你问题的关系。`

风险态：

- 标题：`这段转述太接近原始摘要`
- 说明：`试着改成你自己的判断，而不是换一种说法重复 NotebookLM 的结论。`
- 主按钮：`继续改写`

通过态：

- 标题：`这条转述已经足够形成文献笔记`
- 说明：`你现在可以保存为文献笔记，或继续生成原创笔记候选。`
- 主按钮：
  - `保存为文献笔记`
  - `生成原创候选`

### 4.6 原创候选页

默认态：

- 标题：`原创笔记候选`
- 说明：`这是基于你的转述生成的结构骨架，不是系统替你写好的结论。`

风险态 1：

- 标题：`这条判断还太贴近来源文本`
- 说明：`建议补充“为什么成立”和“边界条件”，再决定是否保存。`

风险态 2：

- 标题：`这条判断缺少证据支撑`
- 说明：`至少绑定一条文献转述或来源证据，再进入原创笔记。`

通过态：

- 标题：`这条候选已经接近可保存`
- 说明：`你可以保存为原创笔记，或再补充边界/反例。`
- 主按钮：`保存为原创笔记`

### 4.7 保存成功页

成功态：

- 标题：`原创笔记已保存`
- 说明：`下一步不要停在这里。把它接进你的主题结构，价值才会继续长出来。`
- 主按钮：
  - `查找可连接笔记`
  - `加入主题索引`
  - `放入写作项目`
  - `继续加工下一条`

## 5. CTA 文案建议

建议避免这些文案：

1. `一键生成原创笔记`
2. `AI 帮我写完`
3. `自动总结并保存`

建议优先使用这些文案：

1. `解析为候选`
2. `加入转述区`
3. `保存为文献笔记`
4. `生成原创候选`
5. `保存为原创笔记`
6. `查找可连接笔记`
7. `加入主题索引`
8. `放入写作项目`

## 6. 设计交付最小要求

如果要交给设计或前端原型，最少应交付：

1. 7 个低保真页面。
2. 每页的空态、正常态、风险态。
3. 至少 1 条从 `NotebookLM summary -> 文献转述 -> 原创候选 -> 保存成功` 的完整演示流。

## 7. 一句话总结

这套界面不是为了让用户“更顺地导入摘要”，而是为了让用户更顺地把摘要推进成自己的原创判断。
