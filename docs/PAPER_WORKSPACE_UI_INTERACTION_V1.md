# 论文工作台 UI 与交互方案 V1

## 1. 文档目标

本方案用于把 `NotebookLM 辅助论文工作流` 从理念层推进到页面与交互层。

它回答的问题是：

1. 页面应该长什么样。
2. 用户每一步应该看到什么、做什么。
3. 系统应该在哪些节点做引导、拦截和提醒。
4. 如何避免产品退化为“摘要搬运器”。

这份文档默认建立在 [NOTEBOOKLM_ASSISTED_PAPER_WORKFLOW_V1.md](/E:/Projects/Thinking%20in%20Notes/yansilu/docs/NOTEBOOKLM_ASSISTED_PAPER_WORKFLOW_V1.md) 的基础上。

## 2. 设计目标

这套界面不追求“一屏展示所有能力”，而追求三件事：

1. 让用户明确知道自己正处在哪个阶段。
2. 让用户持续把外部整理结果推进为自己的判断。
3. 让用户在不被复杂配置打断的情况下，完成从论文到永久笔记的转换。

## 3. 核心原则

### 3.1 不做导入弹窗思维

论文工作台不应被设计成一个短暂弹窗。

它更像一个可持续加工的工作区，用户可能会反复回来补充：

1. 来源材料。
2. NotebookLM 摘要。
3. 自己的转述。
4. 永久笔记候选。
5. 关联和问题。

### 3.2 不让摘要越级

NotebookLM summary / Q&A / note 不应直接越级变成永久笔记。

界面上必须始终清楚区分：

1. 外部整理结果。
2. 我的转述。
3. 我的原创判断。

### 3.3 用流程推进用户，不用一次性表单压住用户

用户不应一开始就看到一大堆字段。

更好的方式是按阶段解锁：

1. 先绑定来源。
2. 再拆候选。
3. 再转述。
4. 再生成永久笔记候选。
5. 最后进入关联与洞见。

## 4. 页面结构

建议页面名称：`论文工作台`

建议采用四栏布局。

### 4.1 左栏：来源区

职责：

1. 展示论文 Source。
2. 展示 PDF / DOI / arXiv / URL。
3. 展示作者、年份、标签。
4. 展示与该论文绑定的 NotebookLM 输入来源。
5. 显示当前工作流阶段进度。

建议内容：

1. 论文标题。
2. 来源元数据卡片。
3. “打开 PDF / 打开原文链接”。
4. “添加 NotebookLM 摘要/问答/笔记”。
5. “查看该论文已生成多少文献笔记 / 永久笔记”。

### 4.2 中左栏：外部整理结果区

职责：

1. 展示 NotebookLM summary / Q&A / study guide / notes。
2. 把大文本拆成可选择的候选卡片。
3. 让用户决定哪些值得继续加工。

建议内容：

1. 来源切换标签：
   - Summary
   - Q&A
   - Study Guide
   - Notes
2. 候选卡片列表：
   - 核心论点
   - 方法
   - 结果
   - 局限性
   - 问题
   - 可引用片段
3. 卡片操作：
   - 加入转述区
   - 标记无用
   - 合并候选
   - 查看原始文本

### 4.3 中右栏：我的转述区

职责：

1. 承接用户自己的理解。
2. 把外部内容变成自己的文献笔记。
3. 形成后续永久笔记候选的基础。

建议内容：

1. 当前正在加工的候选卡片。
2. 转述输入框。
3. 引导问题区。
4. 可选的“我同意/不同意/不确定”立场切换。
5. “这和我已有哪条笔记有关”输入。

建议默认问题：

1. 我怎么理解这点？
2. 这和我正在研究的问题有什么关系？
3. 它成立的条件是什么？
4. 它最值得保留的不是信息，而是哪一个判断？

### 4.4 右栏：永久笔记候选与关联区

职责：

1. 在用户完成转述后，展示永久笔记候选骨架。
2. 展示可连接的已有永久笔记。
3. 展示可能的支持/冲突/补充关系。

建议内容：

1. 永久笔记候选卡片。
2. 结构字段：
   - 核心判断
   - 为什么成立
   - 边界/反例
   - 来源证据
3. 关联建议：
   - 支持
   - 冲突
   - 补充
   - 待比较
4. 操作：
   - 保存为永久笔记
   - 暂存为候选
   - 送入主题索引
   - 送入写作项目

## 5. 顶部全局区域

页面顶部建议保留一条轻量流程栏，而不是传统大表单头部。

流程栏建议包含：

1. 来源已绑定
2. 候选已拆分
3. 转述进行中
4. 永久笔记候选已形成
5. 可进入关联/写作

这样用户始终知道自己是在：

1. 收集材料
2. 理解材料
3. 形成判断
4. 进入结构

## 6. 关键流程

### 6.1 流程 A：从论文到候选

1. 用户创建或打开一个论文 Source。
2. 用户粘贴 NotebookLM summary / Q&A / notes。
3. 系统自动拆出候选卡片。
4. 用户选择值得继续加工的候选。
5. 被选中的候选进入转述区。

通过标准：

1. 用户不用手工切碎长摘要。
2. 用户能快速排除无价值候选。
3. 系统不会把长摘要直接落成单条笔记。

### 6.2 流程 B：从候选到文献转述

1. 用户选中一条候选卡片。
2. 系统显示引导问题。
3. 用户填写自己的转述与立场。
4. 系统保存为文献笔记草稿。
5. 当转述达到最小标准后，才允许下一步。

最小标准建议：

1. 至少有一段用户自己的转述。
2. 至少有一个“与我问题的关系”字段。
3. 对高价值候选，至少有一个边界/保留条件。

### 6.3 流程 C：从文献转述到永久笔记候选

1. 系统根据用户转述生成永久笔记候选骨架。
2. 用户修改核心判断。
3. 系统展示依赖的来源证据。
4. 系统提示潜在边界、冲突或缺口。
5. 用户确认保存为永久笔记。

通过标准：

1. 候选是“结构骨架”，不是大段自动代写正文。
2. 用户能看到这条原创判断依赖了哪些证据。
3. 用户知道它和已有笔记的关系。

### 6.4 流程 D：进入关联与洞见

1. 永久笔记保存成功。
2. 系统立即展示：
   - 可关联的旧笔记
   - 支持关系
   - 冲突关系
   - 可放入的主题索引
3. 用户决定是否继续进入图谱或写作项目。

## 7. 状态设计

每个阶段都应提供最少四种状态：

1. 空态
2. 加载态
3. 正常态
4. 风险态/阻断态

### 7.1 空态

空态不应写成“暂无数据”。

更适合写成下一步动作提示，例如：

1. “先绑定一篇论文来源，再粘贴 NotebookLM summary。”
2. “先挑出一条值得加工的候选，再开始转述。”
3. “先写出你的理解，系统才会帮助生成永久笔记候选。”

### 7.2 风险态

这里的风险提示不只是技术错误，还包括认知风险：

1. 只有摘要，没有转述。
2. 转述太接近原始 NotebookLM 文本。
3. 候选没有落到用户自己的研究问题。
4. 永久笔记候选缺少边界或证据。

风险态文案应该偏“引导”，不应只报错。

## 8. 系统提醒与拦截

### 8.1 允许提醒的地方

可以主动提醒：

1. “这条内容还只是外部整理结果，建议补充你的理解。”
2. “这条转述已经可以尝试生成永久笔记候选。”
3. “这条永久笔记候选和你已有两条笔记存在潜在冲突。”
4. “这篇论文已经形成 3 条永久笔记，可以考虑进入主题索引。”

### 8.2 必须拦截的地方

必须拦截：

1. 用户试图把纯 NotebookLM 文本直接保存为永久笔记。
2. 永久笔记候选没有任何用户自己的转述依据。
3. 永久笔记候选与来源文本过度相似。
4. 永久笔记候选没有证据映射却试图进入写作项目。

## 9. 交互细节建议

### 9.1 候选卡片

每张候选卡片建议至少显示：

1. 类型标签：论点 / 方法 / 结果 / 局限 / 问题。
2. 一段简短摘要。
3. 来源标记：Summary / Q&A / Note。
4. “加入转述区”按钮。
5. “查看原文”入口。

### 9.2 转述编辑区

建议不要只给一个大文本框。

更好的形式是：

1. 上方显示外部候选原文。
2. 下方显示“我的转述”输入区。
3. 右侧或底部显示引导问题 chips。
4. 自动保存草稿。

### 9.3 永久笔记候选卡片

永久笔记候选卡片不应看起来像已经完成的笔记。

需要明显显示：

1. 这是候选，不是已完成。
2. 它依赖哪些文献转述。
3. 哪些字段仍然空缺。
4. 哪些地方有原创性风险。

## 10. 与现有产品结构的关系

这套论文工作台不应替代当前结构，而应接到现有对象上：

1. Source：承接论文和 NotebookLM 输入。
2. Literature Note：承接转述后的文献笔记。
3. Permanent Note：承接用户确认后的原创判断。
4. Index / Graph / Writing Project：承接后续结构化工作。

也就是说，它是“进入主系统的专用入口”，不是另一套独立系统。

## 11. 当前不做

这版交互方案不默认包含：

1. NotebookLM 官方 API 直连。
2. 双向同步。
3. 自动代写永久笔记正文。
4. 自动生成完整研究综述终稿。
5. 重型协作或评论流。

## 12. 最小可演示原型

如果只做一个最小可演示版本，我建议只保留：

1. 左栏论文 Source。
2. 中左 NotebookLM 文本粘贴与候选拆分。
3. 中右转述区。
4. 右栏永久笔记候选。
5. 一个“保存为永久笔记”动作。

这个版本就已经足够验证最关键的问题：

用户是否真的能从 `NotebookLM 辅助阅读` 进入 `自己的永久笔记`。
