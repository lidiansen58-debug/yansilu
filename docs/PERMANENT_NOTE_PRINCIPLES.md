# 永久笔记原则

## 1. 定义

研思录里的“永久笔记（Permanent Note）”不是普通笔记，也不是一段被保存下来的内容。它是一个未来还能被重新理解、重新连接、重新写作的最小思想单元。

永久笔记真正重要的不是“记录了什么”，而是它是否已经成为可以被用户承担、复用和组织的思想节点。

## 2. 成立条件

一条永久笔记必须同时满足以下条件：

1. 只表达一个核心思想。
2. 使用用户自己的语言重写。
3. 单独打开时可以独立阅读。
4. 能进入关系网络、主题索引或写作篮。
5. 能在未来被引用、组合、重构，并长成写作的一部分。

如果一条内容只是摘录、摘要、AI 改写、材料堆叠或一段长文，它还不是永久笔记。

## 3. 产品规则

永久笔记应像一个观点、判断、问题、方法、观察或论证节点，而不是一篇完整文章。

系统在创建、保存、导入、转化和写作时，应默认检查：

1. 这条笔记是否只讲一个思想。
2. 它是否已经从来源文本转化为用户自己的理解。
3. 它是否离开原始上下文也能被理解。
4. 它是否有明确的连接理由、主题归属或后续处理位置。
5. 它是否能作为写作材料进入结构，而不是只停留在存档里。

## 4. AI 边界

AI 可以帮助用户发现候选结构、追问盲点、提示关系和生成写作脚手架，但不能把摘录、摘要或 AI 文本静默保存成永久笔记正文。

永久笔记正文必须由用户确认并承担。系统可以提供骨架、问题和候选关系，但最终的核心表达必须来自用户自己的理解、判断和组织方式。

## 5. 验收标准

任何涉及永久笔记的功能都应通过以下验收问题：

1. 是否避免把多件事塞进一条永久笔记。
2. 是否避免把来源摘录或 AI 改写伪装成用户思想。
3. 是否让笔记在未来脱离原上下文后仍然可读。
4. 是否要求或鼓励用户说明关系、理由和位置。
5. 是否服务于后续写作，而不是服务于资料囤积。
