# 永久笔记优先产品改版设计

## 1. 文档目标

本文档用于修正研思录当前产品设计重心。

新的产品判断是：

`永久笔记是主对象，随笔笔记和文献笔记只是通向永久笔记的辅助材料。`

因此，产品不应让用户感觉自己在维护三套并列笔记系统。

用户应清楚感到：

1. 随笔用于捕捉线索。
2. 文献用于保存证据和转述。
3. 永久笔记用于沉淀自己的判断。
4. 关联让永久笔记产生洞见。
5. 主题索引把关联笔记组织成可写结构。
6. 写作项目和复杂作品从成熟主题继续生长。

---

## 2. 核心产品修正

旧的隐含结构容易被理解为：

`随笔笔记 / 文献笔记 / 永久笔记 三类并列`

新的结构必须被理解为：

`素材入口 -> 永久笔记 -> 关系网络 -> 主题索引 -> 写作项目 -> 复杂作品`

其中：

1. `素材入口` 包括随笔、文献、导入候选、论文工作台等。
2. `永久笔记` 是唯一的一等知识单元。
3. `关系网络` 只围绕永久笔记形成洞见。
4. `主题索引` 是组织多条永久笔记及其关系的主题对象。
5. `写作项目` 只接受永久笔记或展开后的主题索引。
6. `复杂作品` 是多个主题、写作项目和永久笔记长期演化后的结果。

---

## 3. 当前实现基础

当前代码已经具备一部分可复用基础，不需要推倒重来。

## 3.1 已经实现或部分实现

1. 默认启动目录已指向永久笔记盒。
2. 写作篮候选只从永久笔记目录或永久笔记中筛选。
3. 写作篮校验会拒绝非永久笔记。
4. 导入结果里的永久笔记可以一键加入写作篮、打开写作中心或创建写作项目。
5. 主题索引可以作为写作入口，并展开为永久笔记进入写作篮。
6. 随笔和文献笔记可以通过“记录永久笔记”生成永久笔记。
7. 生成永久笔记后，来源笔记会被标记为已关联永久笔记。
8. 原创性守卫只在永久笔记保存时成为强约束。
9. 文献工作区当前已经被统一编辑器替代，复杂表单没有作为默认编辑面板持续暴露。
10. 图谱已经支持当前目录范围、关系统计、支持/冲突/桥接缺口等基础结构信号。

## 3.2 当前主要偏差

1. 导航上三类笔记仍然像并列卡片盒。
2. “新建笔记”会受当前目录影响，主行动不够明确指向“新建永久笔记”。
3. 文献笔记的产品文档仍然要求较重字段，容易把用户拉回资料管理。
4. 随笔和文献的目标状态不够统一：它们都应被看作“是否已经生成原创判断”。
5. 永久笔记详情页还没有完整承载提纯、关联、主题、写作位置四个核心状态。
6. 主题索引和写作中心已经有能力，但入口上仍偏工具区，不够像“永久笔记生长后的下一站”。

## 4. 新信息架构

## 4.1 顶层导航

建议从“模块并列”调整为“永久笔记生长路径”：

1. `永久笔记`
2. `素材入口`
3. `关系网络`
4. `主题索引`
5. `写作与作品`
6. `导入`
7. `设置`

其中：

1. `永久笔记` 是默认首页和默认工作台。
2. `素材入口` 收纳随笔、文献、导入后待处理材料。
3. `关系网络` 只围绕永久笔记显示局部关系和洞见。
4. `主题索引` 组织多个原创判断及其关系。
5. `写作与作品` 从主题索引或永久笔记进入，不从空白 prompt 进入。

## 4.2 左侧目录

仍保留三棵默认目录，但视觉层级要变化：

1. 永久笔记盒：默认展开，一级主区。
2. 素材入口：折叠组，内部包含随笔卡片盒和文献卡片盒。
3. 导入历史和论文工作台：作为素材入口的扩展入口，不与永久笔记盒并列抢主视觉。

## 4.3 主行动按钮

全局主按钮应默认是：

`新建永久笔记`

在素材入口中，主按钮可以变成：

1. 随笔中：`记录永久笔记`
2. 文献中：`从转述记录永久笔记`
3. 导入结果中：`确认永久笔记候选`

但按钮语义始终指向：

`形成自己的判断。`

---

## 5. 三类笔记的新角色

## 5.1 永久笔记

永久笔记是核心对象。

它必须承载：

1. 核心判断。
2. 为什么成立。
3. 边界、反例或反方。
4. 来源证据入口。
5. 关系与连接理由。
6. 所属主题索引。
7. 写作篮和写作项目状态。

永久笔记详情页应回答：

1. 我现在判断了什么？
2. 这个判断凭什么成立？
3. 它和哪些判断有关？
4. 它在哪个主题里？
5. 它是否已经能服务写作？

## 5.2 随笔笔记

随笔不是长期知识单元。

它的状态只有三类：

1. `待处理`：还只是线索。
2. `可记录永久笔记`：已经出现明确判断。
3. `已生成永久笔记`：已沉淀到永久笔记。

随笔界面不需要重型结构。

默认只保留：

1. 快速记录正文。
2. 一句“这里可能有什么判断？”
3. `记录永久笔记` 按钮。
4. 已生成永久笔记后的回跳入口。

## 5.3 文献笔记

文献笔记不是第二套知识库。

它的任务是：

1. 保留来源证据。
2. 逼用户完成转述。
3. 支撑永久笔记的判断。

文献笔记状态建议：

1. `待转述`：只有摘录或来源。
2. `已转述`：用户用自己的话说明了材料。
3. `可记录永久笔记`：转述中已经出现可复用判断。
4. `已生成永久笔记`：已经沉淀到永久笔记。

文献笔记不应作为写作篮输入。

写作需要文献时，应通过永久笔记引用它，而不是直接把文献笔记塞进写作结构。

---

## 6. 文献笔记界面改版

## 6.1 当前问题

文献笔记如果默认展示完整引用字段，会让用户感觉自己在填资料库表单。

这会带来三个问题：

1. 进入成本高。
2. 用户容易停在“资料保存完成”。
3. 产品重心从原创判断滑回文献管理。

## 6.2 新默认界面

文献笔记默认只显示四块：

1. `摘录或材料`
2. `我的转述`
3. `它可能支持的判断`
4. `记录永久笔记`

其中：

1. `摘录或材料` 可以长，但不是完成目标。
2. `我的转述` 是必须动作。
3. `它可能支持的判断` 是通向永久笔记的桥。
4. `记录永久笔记` 是主按钮。

## 6.3 来源信息渐进披露

完整引用字段不要默认展开。

默认显示为一行来源摘要：

`来源：作者 / 年份 / 标题 / 定位`

点击后展开 `来源详情`：

1. 作者。
2. 年份。
3. 标题。
4. 容器信息。
5. 页码、章节、段落或时间戳。
6. URL、DOI、ISBN、PDF 路径等追溯标识。
7. 版本、译者、编者等高级字段。

规则：

1. 进入永久笔记前，必须至少有转述。
2. 进入正式引用或写作证据前，再强提示补足来源定位。
3. 不在用户刚捕捉材料时就阻断所有字段。

## 6.4 文献队列

文献队列不应叫“文献管理”。

建议叫：

`待转永久笔记`

队列分组：

1. `待转述`
2. `可记录永久笔记`
3. `已生成永久笔记`

主指标：

1. 可记录永久笔记数。
2. 已生成永久笔记数。
3. 还缺转述数。

不要把“文献总数”作为主要指标。

---

## 7. 永久笔记详情页改版

永久笔记详情页建议变成四区：

## 7.1 中央：正文与判断

保留 Markdown 编辑器，但右侧或顶部提供轻量状态：

1. 提炼中。
2. 已形成判断。
3. 已接入网络。
4. 已进入主题。
5. 可进入写作。

## 7.2 右侧：思想状态面板

面板包括：

1. 一句话论点。
2. 三句话压缩。
3. 边界/反例。
4. 原创性状态。
5. 作者确认状态。

## 7.3 右侧第二层：关系面板

显示：

1. 当前节点状态。
2. 孤岛处理。
3. 支持、冲突、补充、限定、反例等关系。
4. 连接理由。
5. 洞见候选。

## 7.4 底部或右侧：下一站

显示：

1. 加入主题索引。
2. 加入写作篮。
3. 从当前笔记创建主题。
4. 查看本地图。

原则：

`永久笔记不是写完就结束，而是被放入关系、主题和写作结构。`

---

## 8. 关联与主题路径

产品生长路径建议固定为：

```text
永久笔记
  -> 建立关联
  -> 形成洞见候选
  -> 进入主题索引
  -> 主题成熟度提高
  -> 进入写作篮
  -> 生成脚手架
  -> 沉淀为作品
```

这意味着：

1. 关联功能只围绕永久笔记成为主流程。
2. 文献和随笔可以被引用，但不作为关系网络的主节点。
3. 主题索引必须优先展示永久笔记和关系结构。
4. 写作准备度来自主题下永久笔记的数量、关系质量、反方和来源覆盖。

## 9. 改版实施切片

## P0：不改底层，先改产品重心

1. 全局主按钮改为 `新建永久笔记`。
2. 左侧导航把随笔/文献折入 `素材入口`。
3. 文献笔记默认界面改为四块：摘录、转述、支持判断、记录永久笔记。
4. 完整来源字段移入折叠的 `来源详情`。
5. 随笔和文献列表增加 `已生成永久笔记`、`可记录永久笔记` 状态。
6. 永久笔记详情页强化 `记录关系 / 加入主题 / 加入写作篮` 三个下一步。
7. 首页主指标只保留永久笔记、已接入网络、成熟主题、可写主题。

## P1：把永久笔记做厚

1. 永久笔记提纯面板：一句话论点、三句话压缩、边界/反例。
2. 当前节点关系面板：孤岛、支持、冲突、限定、反例。
3. 保存后关联时刻。
4. 从永久笔记创建主题索引。
5. 主题成熟度。

## P2：进入复杂作品

1. 主题索引升级为作品前结构。
2. 多主题作品工作台。
3. 长期作品状态：seed、structuring、drafting、revising、published。
4. 作品依赖图：主题、永久笔记、来源证据、草稿版本。

## 10. 当前实现对应改法

## 10.1 可以直接复用

1. `record-original-from-note` / `create-original-from-literature` 流程。
2. 写作篮只接受永久笔记的校验。
3. 导入后永久笔记进入写作篮或写作项目的入口。
4. 主题索引进入写作篮的入口。
5. 统一 Markdown 编辑器。
6. 原创性守卫浮窗。
7. 图谱当前目录视图。

## 10.2 需要轻改

1. `新建笔记` 默认改成 `新建永久笔记`，同时保留在当前目录新建的次级操作。
2. 导航和目录命名从三类并列改成 `永久笔记盒 + 素材入口`。
3. 文献队列文案从 `待转述 / 待提炼 / 可转永久笔记` 改成更明确的 `待转永久笔记` 工作流。
4. 搜索和候选列表进一步把永久笔记排到第一位。
5. 编辑器 helper 在没有打开笔记时，不提示“从随笔、文献或永久笔记任选”，而提示“从一条永久笔记开始，或从素材记录永久笔记”。

## 10.3 需要新增

1. 永久笔记状态面板。
2. 关联时刻面板。
3. 孤岛处理面板。
4. 主题成熟度。
5. 文献来源详情折叠 UI。

---

## 11. 验收标准

1. 用户首次进入产品，主视觉默认指向永久笔记，而不是三类笔记并列。
2. 用户能从随笔或文献快速生成永久笔记。
3. 用户不会被文献引用字段在第一屏压住。
4. 文献笔记完成感来自“已转述 / 已生成永久笔记”，不是“字段填满”。
5. 写作篮和写作中心只展示永久笔记或主题索引展开后的永久笔记。
6. 永久笔记详情页能自然引导用户进入关联、主题和写作。
7. 首页和工作台指标默认展示永久笔记、关系、主题和写作准备度。
8. 用户能理解：研思录不是管理资料，而是把资料变成自己的判断网络。

---

## 12. 禁止事项

1. 不把随笔、文献、永久笔记设计成同等重量的三个主产品。
2. 不把文献笔记默认做成复杂引用表单。
3. 不让文献笔记直接进入写作篮。
4. 不把导入量、摘录量、文献总数作为首页主 KPI。
5. 不让 AI 自动把文献或随笔生成并保存为永久笔记。
6. 不在永久笔记尚未确认时，把它用于正式写作结构。

---

## 13. 一句话版本

研思录的产品主线应从：

`管理三类笔记`

改成：

`把素材不断转化为永久笔记，再把永久笔记连接成主题和作品。`
