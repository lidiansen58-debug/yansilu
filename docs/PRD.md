# 研思录 PRD v1.2（内部产品原则版）

## 1. 文档目标

本文档用于统一研思录团队在产品、设计、研发与 AI 编排中的判断边界。
它不是宣传材料，而是核心取舍依据。

---

## 2. 产品定义

研思录是一款面向学习、研究与写作的思维工具。
它的目标不是提高记录效率，而是提高思想形成效率与原创表达质量。

核心链路：

`资料输入 -> 书摘笔记 -> 原创笔记 -> 索引与连接 -> 图谱思考 -> 写作篮 -> 提示词与提纲 -> 原创成文`

任何新增功能都必须回答：

1. 是否强化这条主链路。
2. 是否保留用户原创性与内容主权。

---

## 3. 产品边界（v1.1）

### 3.1 做什么

1. 本地目录工作区（UI 统一称“目录”）。
2. 三类笔记流转：随笔记录、书摘笔记、原创笔记。
3. 原创笔记编辑与关联（`[[标题]]` + 候选搜索）。
4. 当前目录范围内的图谱关系查看（MVP）。
5. 写作中心：选择标签或原创笔记，生成提示词与提纲。
6. Markdown/Obsidian 基础导入导出。

### 3.2 不做什么

1. AI 一键代写原创笔记正文并自动保存。
2. 导入资料后一键生成完整终稿。
3. 通用项目管理、任务管理、社交流。
4. 首页默认打开全量全局图谱。

---

## 4. 关键用户对象与核心价值

1. 学习者与研究者：把外部材料转成可复用观点。
2. 写作者：让文章从积累中生长，而不是从空白页硬写。

核心价值：

1. 从“收集材料”走向“形成观点”。
2. 从“零散笔记”走向“结构化可写”。
3. 从“临时产出”走向“可追溯原创写作”。

---

## 5. 信息架构与术语

1. Vault 是技术名，界面统一叫“目录”。
2. 默认目录树：随笔目录、书摘目录、原创目录。
3. 可在默认目录下新建/隐藏/删除子目录。
4. 新建子目录默认最大笔记数为 `500`。
5. 仅原创目录中的原创笔记允许进入写作流程。

---

## 6. 核心流程

### 6.1 随笔记录

1. 支持文本或语音。
2. 超过 7 天默认隐藏。
3. 可转换为书摘笔记或原创笔记。
4. 转换后原随笔自动归档。

### 6.2 书摘笔记

1. 字段最小集：作者、出版年份、书名、出版社、页码、版本信息（可选）、译者/编者（可选）、摘录、转述。
2. 可长期独立存在。
3. 可发起“新增原创笔记”。
4. 从书摘新增原创后，书摘保持原状态，仅建立关联。

### 6.3 原创笔记

1. 必填结构：核心观点、为什么成立。
2. 标题取 Markdown 第一行。
3. 关联以 `[[标题]]` 显示，内部关系绑定 `note_id`。
4. 标签以正文 `#标签` 解析。
5. 相似度 `>= 80%` 时，阻止保存文件。

### 6.4 索引与图谱

1. 索引笔记是特殊原创笔记。
2. 原创笔记净新增每 15 条触发索引整理提醒。
3. 图谱 MVP 仅展示当前目录中的链接关系。

### 6.5 写作中心（MVP）

1. 输入：标签或原创笔记。
2. 输出：提示词与提纲。
3. 暂不输出完整初稿作为默认结果。

---

## 7. AI 边界

### 7.1 AI 允许

1. 分析资料。
2. 建议索引归属与连接关系。
3. 提示冲突、反例与缺口。
4. 生成写作提示词与提纲。

### 7.2 AI 禁止

1. 直接写并保存原创笔记正文。
2. 自动把外部摘录转成原创笔记。
3. 默认输出替代用户创作的终稿。

### 7.3 AI 输出要求

1. 结构化输出必须走 schema 校验。
2. 结果可解释、可追溯、可拒绝。

---

## 8. 数据与技术原则

1. 目录树、索引笔记、所有笔记均以 Markdown 持久化。
2. SQLite 保存关系与检索结构：链接、标签、反向链接、索引映射、搜索加速。
3. `[[标题]]` 在正文保持可读，SQLite 绑定真实 `note_id`。
4. 缓存可重建：图谱缓存、搜索索引、向量索引。
5. 跨平台路线：`Tauri + React/TypeScript + SQLite + CodeMirror 6`。

---

## 9. 成功标准（MVP）

1. 用户能从资料形成书摘笔记。
2. 用户能从书摘笔记形成原创笔记。
3. 用户能通过索引与连接建立结构。
4. 用户能在当前目录图谱中看到可写路径。
5. 用户能基于原创笔记生成提示词与提纲。

---

## 10. 决策检查清单

每次评审新功能必须回答：

1. 是否帮助用户从资料走向原创笔记。
2. 是否帮助用户建立结构而非制造噪音。
3. 是否帮助用户从积累走向写作。
4. 是否保留用户原创性。
5. 是否把产品推向“泛化笔记工具”。

若答案不清晰，不进入核心版本。
