# 关联笔记图谱思考工作台 UX 方案

Updated: 2026-06-04

## 1. 设计目标

关联笔记图谱不是用来证明用户“有很多连接”，而是帮助科研人员判断：

- 哪些永久笔记正在形成可研究的主题。
- 哪些笔记孤立、错位、缺少中间判断。
- 哪些关系理由偏弱、方向可疑、类型不准。
- 哪些冲突、边界、反例、意外桥接可能带来新的追问。

默认页面只回答一个问题：

> 这批永久笔记里，哪里最值得我继续思考？

## 2. 用户假设

- 用户是非技术科研人员，不应该先理解图算法、关系类型体系或数据结构。
- 用户主要关心自己的知识是否更清楚，而不是图谱是否复杂。
- 用户需要系统指出“可追问处”，但最终判断仍由用户完成。
- 用户可能在桌面上深度整理，也可能在移动端快速检查一个局部结构。

## 3. 设计原则

1. 默认安静，只显示当前最值得处理的线索。
2. 系统提示使用“可追问处”“待判断”“候选”，避免替用户宣布洞见。
3. 图例、统计、说明、AI 初判细节默认隐藏，按需打开。
4. 节点大小表达当前注意力，不表达最终学术价值。
5. 点击节点读“笔记角色”，点击关系读“为什么相连”，点击簇读“是否成题”。
6. 每一个系统判断都必须能解释依据。
7. 动效只服务聚焦、切换和新线索出现，不做持续装饰。

## 4. 信息架构

默认页面由三层组成：

- 主画布：展示当前范围的主要笔记结构和局部焦点。
- 顶部工具带：只放范围、搜索、图例、全屏等基础控制。
- 按需面板：点击节点、关系、主题簇或可追问处后出现。

不要在默认页面常驻：

- 完整图例。
- 大段读法说明。
- 大块统计卡片。
- AI 图谱初判详情。
- 全量关系维护队列。

## 5. 默认态低保真

```text
+------------------------------------------------------------------------+
| 永久笔记图谱 / 当前范围: 易经研究                         搜索 图例 全屏 |
+------------------------------------------------------------------------+
|                                                                        |
|                         .      o                                       |
|                   o------*-----o      o                                |
|                         /  \                                           |
|              o         o    o-----o                                    |
|                                                                        |
|                                                                        |
|  [5 个可追问处]                                                        |
+------------------------------------------------------------------------+
```

默认画布只呈现图，不常驻解释文字。左下角的 `可追问处` 胶囊是唯一的主动引导。

胶囊文案示例：

- `5 个可追问处`
- `1 个主题候选 · 2 条关系待复核`
- `3 个孤立笔记等待判断`

## 6. 顶部工具带

默认可见控制：

| 控制 | 作用 | 备注 |
| --- | --- | --- |
| 当前范围 | 显示当前目录、主题或当前笔记 | 可点击切换范围 |
| 搜索/聚焦 | 找到一条笔记或主题 | 输入后局部聚焦 |
| 图例 | 打开轻浮层 | 默认隐藏图例 |
| 全屏 | 放大画布 | 桌面优先 |

暂不默认展示：

- `找洞见`
- `看主题`
- `理关系`
- 关系类型下拉
- AI 扫描按钮

这些能力进入右侧思考面板，避免首次进入时先面对模式选择。

## 7. 可追问处面板

点击左下角胶囊后，右侧打开思考面板。

```text
+------------------------------+
| 可追问处                     |
| 全部  主题  整理             |
+------------------------------+
| 主题候选                     |
| 易经中的开放性解释           |
| 主题胚胎 / 缺少反方和边界    |
| [继续追问] [形成索引卡]      |
+------------------------------+
| 关系待复核                   |
| A -> B                       |
| 理由太短，可能只是标题相似   |
| [补理由] [改类型]            |
+------------------------------+
| 孤立笔记                     |
| 慢读让旧观点重新换位         |
| 待判断 / 可能需要桥接        |
| [寻找关联] [移入暂存]        |
+------------------------------+
```

筛选视角：

- `全部`：按优先级列出当前最值得处理的 3-7 个对象。
- `主题`：只看主题候选、主题成熟度和索引卡入口。
- `整理`：只看孤立、弱关系、方向可疑、类型可疑等维护对象。

## 8. 节点选中态

点击节点后，右侧面板显示这条笔记在当前结构中的角色。

```text
+------------------------------+
| 笔记角色                     |
| 慢读让旧观点重新换位         |
+------------------------------+
| 当前判断                     |
| 这条笔记可能是一个桥接候选。 |
|                              |
| 判断依据                     |
| 连接数少，但内容与两个主题相邻。 |
| 与“卦序是理解路径”有潜在过渡。 |
|                              |
| 可追问                       |
| 它是在补充现有主题，还是开启新主题？ |
+------------------------------+
| [寻找关联] [补桥接] [加入写作] |
+------------------------------+
```

节点角色标签：

- `当前焦点`
- `主题核心候选`
- `支撑材料`
- `桥接候选`
- `边界条件`
- `孤立待判断`
- `暂存线索`

## 9. 关系选中态

点击关系线后，显示关系复核面板，不直接跳转源笔记。

```text
+------------------------------+
| 关系复核                     |
| A -> B                       |
+------------------------------+
| 类型                         |
| supports                     |
|                              |
| 关系理由                     |
| A 支撑 B，因为……             |
|                              |
| 为什么需要复核               |
| 理由偏短，尚未说明中间判断。 |
+------------------------------+
| [补理由] [改类型] [暂不采纳] |
| 更多                         |
+------------------------------+
```

关系待复核原因：

- `理由缺失`
- `理由太短`
- `关系类型可能不准`
- `方向可能反了`
- `只是标题相似`
- `与当前主题关系较弱`

`断开关系` 放在更多菜单里，不作为第一层动作。

## 10. 主题簇选中态

点击主题簇或主题候选后，显示主题成熟度。

```text
+------------------------------+
| 主题候选                     |
| 易经中的开放性解释           |
+------------------------------+
| 成熟度                       |
| 主题胚胎                     |
|                              |
| 为什么这样判断               |
| 有中心判断。                 |
| 有 4 条支撑关系。            |
| 缺少反方或边界。             |
| 已出现 1 个可追问问题。      |
+------------------------------+
| 下一步                       |
| 先补一个反例或边界，再形成索引卡。 |
|                              |
| [继续追问] [形成索引卡] [加入写作] |
+------------------------------+
```

主题成熟度：

| 状态 | 含义 | 界面表达 |
| --- | --- | --- |
| `材料堆` | 有相近笔记，但中心判断不清楚 | 低饱和簇轮廓 |
| `主题胚胎` | 有中心判断和支撑，但边界/反方不足 | 柔和高亮簇轮廓 |
| `可形成索引卡` | 中心、支撑、边界、追问基本成立 | 清晰主题边界和索引卡动作 |

主题成熟度依据：

- 是否存在中心判断。
- 是否存在 3 条以上支撑或展开关系。
- 是否存在限定、反方、反例、边界。
- 是否存在可追问问题。
- 是否有桥接到相邻主题的关系。

## 11. 孤立节点管理

孤立节点不默认被判定为错误，而是进入 `待判断`。

孤立节点分类：

| 分类 | 含义 | 动作 |
| --- | --- | --- |
| `待桥接` | 内容有价值，但缺中间关系 | 寻找关联、补桥接 |
| `暂存` | 上下文不足，暂时不处理 | 移入暂存、稍后提醒 |
| `疑似错位` | 可能属于另一个主题或目录 | 移动范围、重新归类 |

孤立节点面板问题：

- 它应该连接到哪个问题？
- 它是在补充现有主题，还是开启新主题？
- 它是否只是材料，还没有形成永久笔记判断？

## 12. 节点大小和视觉权重

节点大小表达当前注意力，不表达最终价值。

| 尺寸 | 用途 |
| --- | --- |
| `XL` | 当前聚焦节点 |
| `L` | 主题核心候选、桥接价值高、关系待复核 |
| `M` | 普通永久笔记 |
| `S` | 边缘线索、暂存节点 |

放大原因必须可解释。悬停或选中节点时，tooltip 或面板应显示：

- `当前焦点`
- `主题核心候选`
- `桥接价值高`
- `关系待复核`
- `孤立待判断`

## 13. 关系线语义

| 类型 | 视觉表达 |
| --- | --- |
| `支持` | 稳定实线，青绿色 |
| `冲突/反例` | 低饱和玫瑰红，带轻微张力感 |
| `限定/边界` | 琥珀色收束线 |
| `桥接` | 冷蓝或青绿引导线 |
| `候选/待确认` | 半透明虚线 |

不能只依赖颜色表达关系类型。图例和面板里必须有文本标签。

## 14. 图例策略

图例默认隐藏。点击 `图例` 后出现轻浮层。

图例只解释必要内容：

- 节点大小表示当前注意力，不表示最终价值。
- 实线表示已确认关系。
- 虚线表示候选或待确认关系。
- 不同线型对应支持、冲突、限定、桥接。

图例不承担教学文档功能。更长说明进入帮助文档或空状态。

## 15. 移动端方案

移动端默认不展示完整复杂图。

移动端结构：

```text
+----------------------+
| 当前范围             |
| [3 个可追问处]       |
+----------------------+
| 局部图预览           |
| 当前焦点 + 一跳关系  |
+----------------------+
| 可追问处列表         |
| 主题候选 / 孤立 / 复核 |
+----------------------+
```

移动端交互：

- 点击节点：底部 sheet 显示笔记角色。
- 点击关系：底部 sheet 显示关系复核。
- 点击 `打开全图`：进入全屏图谱。
- 不依赖 hover。
- 所有可点区域至少 44px。

## 16. 可访问性要求

- 节点、关系、主题簇都必须可键盘聚焦。
- 图例按钮、全屏按钮、面板关闭按钮必须有清晰标签。
- 关系线不能只靠颜色区分，必须有文本标签或可读说明。
- 面板打开后焦点进入面板，关闭后回到触发对象。
- `Esc` 关闭图例、面板或全屏状态。
- 支持 `prefers-reduced-motion`，关闭持续动画和非必要转场。

## 17. 视觉方向

风格关键词：

- 安静的研究星图。
- 克制、轻、清楚、有响应。
- 内容优先，科技感来自空间层次和局部反馈。

视觉 token 建议：

| Token | 建议 |
| --- | --- |
| 背景 | 浅雾白、冷灰、极淡网格 |
| 主色 | 青绿 + 冷蓝 |
| 提醒色 | 琥珀 |
| 冲突色 | 低饱和玫瑰红 |
| 焦点 | 柔和 halo |
| 候选 | 半透明虚线 |
| 面板 | 轻量抽屉，8px radius，低阴影 |

动效只用于：

- 进入焦点。
- 打开或关闭面板。
- 新的可追问处出现。
- 从一个局部切到另一个局部。

不使用持续闪烁、大面积流光或强装饰光效。

## 18. 现有元素去留映射

| 现有元素 | 处理 | 原因 |
| --- | --- | --- |
| 顶部图谱标题 | 保留并压缩 | 需要范围感，但不应占太多空间 |
| 图谱摘要长句 | 改为短范围状态 | 长句增加认知负担 |
| 刷新图谱按钮 | 保留，放入工具菜单或次要按钮 | 需要手动刷新，但不应成为主视觉 |
| 论证图/结构图切换 | 移入思考面板筛选 | 默认不让用户先选模式 |
| 关系类型下拉 | 移入高级筛选 | 常驻会让页面偏技术化 |
| 大画布 | 保留并强化 | 主体验仍是图 |
| hover 说明卡 | 改为按需面板或轻提示 | 常驻解释抢主视线 |
| 完整图例 | 隐藏到图例浮层 | 需要时查看 |
| 指标卡片 | 合并进可追问处胶囊和面板 | 统计不应压过思考 |
| 图谱洞见卡 | 改名为可追问处 | 避免系统替用户宣布洞见 |
| 桥接缺口列表 | 移入思考面板 | 属于可追问处 |
| 待补关系理由 | 移入整理筛选 | 属于关系复核 |
| AI 图谱初判 | 移入面板或更多操作 | 默认不占第一屏 |
| 节点点击打开笔记 | 改为先打开笔记角色面板 | 图谱页先解释结构 |
| 边点击打开源笔记 | 改为打开关系复核面板 | 用户点边是想读关系 |

## 19. 实现切片建议

### Slice 1: 信息层级收敛

- 简化顶部工具带。
- 新增 `可追问处` 胶囊。
- 默认隐藏图例、指标、AI 初判和维护队列。
- 保留现有图谱数据和布局逻辑。

验收：

- 默认第一屏只出现范围、搜索/工具、画布、可追问处胶囊。
- 图例通过按钮打开。
- 现有关系图谱仍能加载和刷新。

### Slice 2: 右侧思考面板

- 点击可追问处胶囊打开面板。
- 将桥接缺口、关系待复核、主题候选、孤立笔记合并成列表。
- 面板提供 `全部 / 主题 / 整理` 筛选。

验收：

- 用户能从面板进入补理由、补桥接、形成索引卡等动作。
- 面板关闭后回到图谱画布。

### Slice 3: 节点和关系选中态

- 点击节点打开笔记角色面板。
- 点击关系打开关系复核面板。
- 增加关系线键盘聚焦。

验收：

- 点边不再直接打开源笔记。
- 关系复核能显示类型、理由、复核原因和 3 个主动作。

### Slice 4: 主题候选表达

- 增加主题簇成熟度。
- 显示中心判断、支撑、边界、可追问问题。
- 增加主题簇面板。

验收：

- 至少能解释 `材料堆`、`主题胚胎`、`可形成索引卡` 三种状态。

### Slice 5: 视觉和移动端完善

- 加入节点注意力尺寸。
- 增加局部放大和邻域降噪。
- 移动端改为局部图预览 + 底部 sheet。
- 支持 reduced motion。

验收：

- 桌面和移动端都不依赖 hover 完成核心操作。
- 当前焦点的放大原因可解释。

## 20. 验收问题

一版设计可以进入实现前，应能回答：

- 用户进入图谱页 5 秒内能否知道哪里值得继续看？
- 用户能否区分“系统提示”和“自己的判断”？
- 用户点一条关系后，是否能看懂为什么相连？
- 用户能否知道一个主题候选为什么成熟或不成熟？
- 孤立节点是否被表达为待判断，而不是错误？
- 图例和统计是否已经从默认界面退到按需层？
- 移动端是否能不用 hover 完成节点、关系、可追问处查看？
