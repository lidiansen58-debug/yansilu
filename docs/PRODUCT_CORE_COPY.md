# 产品核心文案

这份文档用于统一研思录的核心表达，避免首页、帮助、 Demo、发布文案、官网介绍各说各话。

配套对外中文短稿见：
`docs/PRODUCT_EXTERNAL_COPY_CN.md`

## Slogan

**让笔记长出思想**

## 超短版

记录、重述、关联，让笔记长出思想。

## 标准版

研思录帮助你把零散材料慢慢变成自己的理解。
先记录，再用自己的语言重述，沉淀为永久笔记，建立关联，形成知识网络。
知识网络会持续复利，为后续写作、研究和创新提供基础。

## 帮助页版

研思录不是把记录、整理、关联、写作拆成几个孤立功能，
而是把它们连成一条会不断产出新理解的路径。

你可以从一句随笔开始，
把材料重述成自己的理解，
把成熟判断沉淀为永久笔记，
再通过关联让笔记彼此解释、彼此推动。

当多条永久笔记围绕同一个问题聚集时，
它们会慢慢长成可写主题，
再进入写作中心，变成提纲和草稿。

## Demo 导览版

这套 Demo 想表达的核心不是“记得更多”，而是：

**记录、重述、关联，让笔记长出思想。**

你不用先学完整方法论。
照着导览走一遍，就能看到研思录怎样把
记录 -> 转述 -> 永久笔记 -> 建立关联 -> 主题索引 -> 写作中心
连成一条主路径。

## 首页短说明版

从记录、重述、关联开始，让笔记慢慢长成可写的理解。

## 空库首屏版

第一次打开，不需要先理解全部功能。
先选一件事开始：
- 导入示例库
- 选择已有笔记库
- 写第一条笔记

先开始，再逐步整理。

## 发布说明版

研思录是一款面向长期知识积累与写作的本地优先笔记工具。
它强调用自己的语言重述材料，把成熟判断沉淀为永久笔记，
并通过关联形成可以持续复用的知识网络。

它不是为了囤积资料，
而是为了让笔记长出思想，并为未来写作与创新提供基础。

## 用词约束

优先使用这些表达：

- 记录
- 重述
- 永久笔记
- 关联
- 可写主题
- 写作中心
- 让笔记长出思想

尽量避免在面向新手的主路径里优先使用这些词：

- 候选
- 队列
- 复核
- 线索
- 脚手架
- 请求包
- 诊断层

## 使用建议

- 首页和空库首屏优先用“超短版”或“首页短说明版”
- 帮助文档优先用“帮助页版”
- Demo 说明优先用“Demo 导览版”
- 发布说明、商店描述、官网介绍优先用“发布说明版”
- 如果空间非常小，只保留 slogan 和“超短版”
